    Genie Centurion: Accelerating Scalable Real-World Robot Training with Human Rewind-and-Refine Guidance (2025)¶

    基本信息¶

    • 作者: Wenhao Wang, Jianheng Song, Chiming Liu*, Jiayao Ma, Siyuan Feng, Jingyuan Wang, Yuxin Jiang, Kylin Chen, Sikang Zhan, Yi Wang, Tong Meng, Modi Shi, Xindong He, Guanghui Ren, Yang Yang, Maoqing Yao
    • 机构: AgiBot
    • arXiv: 2505.18793
    • 项目: genie-centurion.github.io

    一句话总结¶

    提出 GCENT 数据收集范式：机器人先部署不完美策略，失败时"倒带"到历史状态并由人类提供修正示范，通过 Task Sentinel 自动成功检测实现单人监督多机器人，数据效率提升 2.25x、任务成功率提升 40%。

    问题¶

    VLA 训练的数据收集瓶颈：

    1. 全程遥操作效率极低：传统遥操作需要人类全程关注，人机比最高 1:1，实际因任务重置/人为失误等开销更低；
    2. 数据质量不足：全轨迹遥操作收集大量冗余的"简单段"（如接近物体），而真正关键的"失效点"（如抓取最后动作块、精密对准）在数据中占比过低；
    3. sim-to-real 转移不稳定：仿真扩展路线在接触丰富的高精度任务中可靠性无保证；
    4. 无法捕获策略执行时的真实失效模式：静态遥操作的失效分布与策略部署时的真实失效分布存在严重偏差（协变量偏移），限制策略收敛效率。

    方法¶

    • 方法线归属: 交互式模仿学习 / DAgger-inspired 数据收集系统（与 RoboCopilot 同属"Human-in-the-loop 交互式数据收集"支线，但聚焦于可扩展的多机器人场景）
    • 核心 idea: "deploy-then-collect"范式——将不完美策略直接部署，通过倒带（Rewind）机制恢复历史状态 + 人类精修（Refine），将收集重心集中于真实失效点；Task Sentinel 自动监控步骤完成状态，使人类可离开全程监控转为按需介入。

    关键技术点¶

    1. Rewind（倒带）机制¶

    • 系统在机器人本体上实时维护过去 3 秒的状态缓冲区
    • 人类发现策略失败时，按 X 键触发倒带：反向发送历史关节状态将机器人恢复到失败前的合法先验状态
    • 倒带后操作者可施加物理扰动，再进行修正示范（Refine）
    • 关键价值：避免了从完全初始态重置，直接在失效点附近的状态空间上收集高价值样本，既提升样本质量又扩大关键状态空间覆盖

    2. Task Sentinel：多模态步骤成功检测¶

    • 基于 InternVL 2.5-2B 骨干 + MLP 二值分类头
    • 输入：当前图像观测 o_t + 任务指令 l_task，输出步骤成功状态 z_t ∈ {0,1}
    • 检测成功而非失败：任务成功的定义清晰稳定，而失败类型随训练阶段动态变化，难以穷举
    • 任务分步标注：将长时序任务分解为子步骤，人工标注每步起止帧，训练成二值分类任务
    • 运作逻辑：z_t=1 → 自动进入下一步；z_t=0 且 Δt > T_max → 进入"等待介入"模式请求人类
    • 随策略改善渐进退场：早期低成功率时以人类直接监控为主，随成功率上升 Task Sentinel 接管常规监控，人类仅处理异常

    3. 四阶段运作循环¶

    初始化: 收集少量种子数据 D₀ → 训练初始策略 π₀

部署迭代 i (循环直到平均分 > 0.9):
  a) Inference: 策略 πᵢ + Task Sentinel 自主执行任务
  b) Monitoring: Task Sentinel 监控每步完成状态
  c) Rewind & Refine: 失败时倒带 → 人类提供修正示范 Drefine
  d) Data Aggregation: 聚合有效轨迹（仅保留 Intervention 段作为高质量样本）

重训练: 用 Dᵢ₊₁ 精调策略 πᵢ₊₁ 和 Task Sentinel Sentinelᵢ₊₁
    - 仅精调干预段：GO-1 基础模型在接近阶段极少失败，因此仅用人类干预段精调"脆弱步骤"，减少计算浪费

    4. 硬件与接口设计¶

    • 基于 AgiBot G01 双臂机器人平台 + VR 双 6-DoF 控制器
    • 四种模式的按键分配：Y（推理）/ X（倒带）/ 侧键（干预遥操作）/ A（重置）
    • 数据管线：本地 HDF5 → 云端帧对齐 + 步骤标注 + 模型训练
    • 自动记录模式标签（Inference / Intervention / Rewind / Awaiting Intervention），"Intervention"段自动识别为高质量训练样本

    5. 倒带策略的阶段性使用¶

    • 早期（策略成功率 ~20%）：直接干预（不倒带）更优——让模型学习从失败中恢复的能力
    • 后期（策略成功率 ~80%）：倒带后再干预更优——清除错误序列、提升首次成功率和轨迹质量
    • 消融实验验证：早期 Direct Intervention 0.69 vs Rewind 0.50；后期 Rewind 0.91 vs Direct Intervention 0.84

    实验¶

    实验设置¶

    • 平台: AgiBot G01 双臂机器人，AgiBot Orin 推理板
    • 基础模型: GO-1 foundation model（ViLLA 架构，100万+轨迹，217任务预训练）
    • 4 个真机任务:
    • Sandwich Assembly（三明治组装）: 8个顺序双臂拣放，长时序
    • Connector Insertion（连接器插入）: 接触丰富、高精度插入
    • Microwave-Heating（微波加热）: 5种不同操作（拉/拾/放/推/按），长时序
    • Typing（键盘打字）: 指令跟随，包含退格处理，精密定位
    • 评分: 分步成功率（步骤完成数/总步骤数，1.0=完全成功）
    • 对比基线: (1) Teleop（传统全轨迹遥操作），(2) ADC（对抗性数据收集，引入人为扰动）

    主要结果（Q1：性能比较）¶

    方法 Sandwich Insertion Microwave Typing 最终平均
    Teleop 0.45 (100 traj) 0.40 (60 traj) 0.55 (80 traj) 0.10 (100 traj) 0.38
    ADC 0.53 (100 traj) 0.82 (60 traj) 0.76 (80 traj) 0.03 (100 traj) 0.53
    GCENT 0.91 (120 traj) 0.90 (60 traj) 0.97 (80 traj) 0.95 (100 traj) 0.93
    • GCENT 在所有任务上均超过 0.9，最终轮相对 Teleop 和 ADC 分别提升约 145% 和 75%（基于原始分数比较）

    数据效率（Q2）¶

    • 相同帧数下：GCENT 平均性能提升 40%
    • 相同性能水平下：GCENT 仅需对比方法 44.5% 的帧数
    • Typing 任务（精密指令跟随）尤为突出：仅需 30k 帧即达 0.9+

    多机器人扩展性（Q2）¶

    策略成功率 干预率 有效帧采集效率
    40% 47% 1.86 (vs 理论最大 2.0)
    60% 39% 1.90
    80% 27% 1.92
    - 即使策略成功率仅 40%，单人双机器人运作效率已达 1.86，证明 Task Sentinel 的核心价值

    倒带策略消融（Q3）¶

    • 早期（起始 20%）：直接干预 0.69 > 倒带 0.50（倒带限制了失败恢复能力学习）
    • 后期（起始 80%）：倒带 0.91 > 直接干预 0.84（倒带提升首次成功轨迹质量）

    评价¶

    优势¶

    1. 系统设计务实，工程落地性强：倒带机制不依赖精密硬件或额外传感器，基于现有关节状态缓冲即可实现，可推广到任意支持双向驱动的平台
    2. Task Sentinel 设计巧妙：检测成功而非失败——避免了失败类别随训练阶段动态变化的难题，训练稳定
    3. 阶段性策略洞察：发现倒带机制在不同训练阶段有相反效果（早期有害/后期有益），揭示了"失败恢复能力"与"优化首次成功轨迹"之间的阶段性权衡
    4. 真实扩展性验证：1.86~1.92 的双机器人效率比有具体测量数据支撑，而非定性声称
    5. 与 GO-1 基础模型深度结合：针对"接近阶段少失败/关键点常失败"的特点，仅精调干预段，高效利用大模型的已有能力

    局限¶

    1. 依赖足够好的基础模型：GCENT 从"不完美策略"出发，但若基础策略质量极差（零成功率），早期迭代效率大幅下降（如 Connector Insertion 起始 0% 时前几轮进展缓慢）
    2. 倒带机制仅限物理上可逆的操作：对于破坏性操作（如洒液体、切割）或接触丰富且不可逆的任务，直接倒带关节状态可能不安全
    3. Task Sentinel 需要标注：每个任务需要人工标注子步骤起止帧，新任务引入存在额外标注成本
    4. 评测协议局限：论文指出分步评分在后期迭代中难以区分细微改进，可能掩盖实际进步
    5. 未探索完全异步或完全自主场景：未来 1-to-N（N>2）的扩展和对负样本的更充分利用（如 post-training with negative examples）被列为未来工作

    与相关工作的定位对比¶

    工作 核心关注 倒带机制 自动监控 扩展性
    RoboCopilot (2024) 无缝人机切换硬件 ❌ ❌（人工全程监控） 单机器人
    Fleet-DAgger (2022) 多机器人干预调度 ❌ 有限 多机器人（调度策略）
    HIL-SERL (2024) RL+人在回路精细操作 ❌ ❌ 单机器人
    RoboClaw (2026) 全生命周期agentic框架 ❌（EAP自重置） VLM元控制器 系统级
    GCENT（本文） 可扩展实采数据收集 ✅（关节状态缓冲） ✅（Task Sentinel） 1:N验证

    核心差异化：GCENT 是当前唯一将倒带机制（状态空间覆盖）+ 自动任务监控（Task Sentinel）+ 多机器人并行监督三者系统集成的真机数据收集框架。

    对 VLA 领域的贡献¶

    1. 验证了"deploy-then-collect"范式在真实机器人操作中的有效性：相比自动驾驶领域成熟应用，这是在接触丰富、精度要求高的机器人操作中最系统的实证之一
    2. 揭示数据质量集中度胜过覆盖广度：失效点的数据比均匀全轨迹数据价值更高，为VLA精调数据设计提供依据
    3. Task Sentinel 提供了实用的运行时策略评估范式：区别于 VQA 式成功检测，reward-model-style 的逐帧二值分类更适合在线监控场景
    4. 为 AgiBot 生态提供了配套的数据飞轮：GCENT 与 GO-1 基础模型结合，形成"预训练大模型 + 在线精调关键步骤"的高效训练闭环，可直接接入 AgiBot World/Colosseo 数据管线
