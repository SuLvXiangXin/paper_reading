    Octo: An Open-Source Generalist Robot Policy (2024)¶

    基本信息¶

    • 作者: Dibya Ghosh, Homer Walke, Karl Pertsch, Kevin Black, Oier Mees*, Sudeep Dasari, Joey Hejna, Tobias Kreiman, Ria Doshi, Charles Xu, Jianlan Luo, You Liang Tan, Lawrence Yunliang Chen, Pannag Sanketi, Quan Vuong, Ted Xiao, Dorsa Sadigh, Chelsea Finn, Sergey Levine
    • 机构: UC Berkeley, Stanford, CMU, Google DeepMind
    • arXiv: 2405.12213
    • 链接: https://octo-models.github.io
    • 开源: ✅ 完全开源（模型检查点、训练代码、数据加载器、微调脚本，JAX实现）

    一句话总结¶

    首个完全开源的跨具身通用机器人策略（Generalist Robot Policy），基于 ViT 风格 transformer + diffusion action head 在 800k 机器人轨迹（Open X-Embodiment 子集）上预训练，通过模块化 tokenizer + block-wise attention 设计支持灵活的输入/输出适配，可高效微调到新的传感器、动作空间和机器人形态。

    问题¶

    已有的通用机器人策略（Generalist Robot Policies, GRPs）如 RT-1-X、RT-2-X、RoboCat 存在三大局限： 1. 输入/输出不灵活：要求下游用户使用预训练时固定的传感器配置和动作空间，无法适配新的观测（如力/力矩传感器）或动作空间（如关节位置控制） 2. 不支持高效微调到新域：添加/移除输入模态时需要重新初始化或重训练大量参数 3. 不开源：最大最强的模型（如 RT-2-X 55B）不公开可用

    核心问题：如何设计一个开源、灵活、可扩展的通用机器人策略，使其既能零样本控制多种机器人，又能高效微调到全新的传感器配置、动作空间和机器人形态？

    方法¶

    • 方法线归属: Transformer + Diffusion Head（Cross-embodiment Generalist Policy）
    • 注意：Octo 不基于预训练 VLM（没有 SigLIP/PaliGemma 等），其 transformer backbone 从头训练
    • 是 VLM + Diffusion/Flow Head 方法线（π₀系列）的重要前身，但架构更简单、规模更小
    • 核心 idea: 用模块化的 token 化设计（任务 token + 观测 token + readout token）+ block-wise masked attention 构建灵活的 transformer 策略，使得微调时可以自由添加/移除输入输出而无需重新初始化预训练权重

    关键技术点:¶

    1. 模块化 Transformer 架构（核心设计创新）¶

    • Input Tokenizers:
    • 语言：t5-base (111M) 冻结编码器 → 16 个语言 token
    • 图像（3rd person / wrist camera）：浅层 CNN → 展平为 patch 序列（256 patches@256×256, 64 patches@128×128）
    • 非 VLM 方式：用轻量 CNN 而非预训练视觉编码器
    • Transformer Backbone: ViT 风格（ViT-S 27M / ViT-B 93M），参数集中在 transformer 而非视觉编码器（"transformer-first"设计，区别于 RT-1/RT-2 的 ResNet 重 + transformer 轻）
    • Block-wise Masked Attention:
    • 观测 token 只能 causal attend 到同一或更早时间步的观测 token + 任务 token
    • 缺失模态的 token 被完全 mask 掉（如无语言标注时 mask 语言 token）
    • Readout token（类似 BERT 的 [CLS]）：可以 attend 到所有先前 token，但不被其他 token attend → 被动读取信息，不影响内部表示
    • 微调灵活性：添加新观测只需新 tokenizer + 位置编码；添加新动作只需新 action head；不修改预训练 transformer 权重

    2. Diffusion Action Head¶

    • DDPM 目标 + cosine noise schedule，20 步去噪
    • 轻量 3 层 MLP（hidden dim=256，带残差连接和 LayerNorm）
    • 关键设计：只需 transformer 前向一次，多步去噪全在小型 diffusion head 内完成
    • 预测 action chunk（多步未来动作）
    • 对比：大幅优于 MSE head（hedging 问题，动作过慢）和 discretized head（缺精度，抓取常失败）

    3. Cross-embodiment 训练数据¶

    • 800k 轨迹，来自 Open X-Embodiment 的 25 个数据集（手动筛选）
    • 比 RT-X 的 350K 更大更多样
    • 数据筛选标准：保留有图像流 + delta end-effector 控制的数据集，移除过于重复/低分辨率/极端 niche 的数据集
    • 数据加权：按数据集大小加权，"更多样"数据集权重翻倍，过于重复的数据集降权
    • 对齐：gripper 动作统一（+1=开, 0=合），缺失相机通道零填充
    • 任务标签：随机零化语言或目标图像以支持两种条件模式；无语言标注的数据集始终用目标图像条件（hindsight goal relabeling）

    4. 高效微调设计¶

    • 微调时更新全部模型参数（实验发现优于冻结子集）
    • 统一微调配方：~100 条目标域轨迹，50k 步，cosine decay + linear warmup
    • 消费级 GPU 可用：单张 NVIDIA A5000 (24GB) 约 5 小时
    • 可适配：新传感器（力/力矩）、新动作空间（关节位置控制）、新机器人形态（双臂）

    架构细节¶

    模型 Layers Hidden MLP Heads 参数量
    Octo-Small 12 384 1536 6 27M
    Octo-Base 12 768 3072 12 93M
    • 语言编码器：t5-base (111M)，冻结
    • 训练：AdamW, lr=3e-4, inverse sqrt decay, batch size=2048, 300k steps
    • 硬件：TPU v4-128 pod，14 小时完成预训练

    实验¶

    A. 零样本多机器人控制（vs RT-1-X, RT-2-X）¶

    • 平台: WidowX, UR5, RT-1 Robot（3 种机器人）
    • 结果: Octo 平均比 RT-1-X (35M) 高 29% 成功率；与 RT-2-X (55B) 性能相当
    • 目标图像 vs 语言：目标图像条件比语言条件高 25% 成功率（目标图像包含更多信息）
    • 泛化分析（BridgeV2 细粒度）：新物体表现好，新场景略降，新技能（翻转、精确插入）显著下降

    B. 高效微调评估（vs from scratch, VC-1）¶

    • 6 个微调场景: Berkeley Insertion（新传感器）、Stanford Coffee、CMU Baking、Berkeley Pick-Up（新动作空间：关节控制）、Berkeley Coke（新机器人）、Berkeley Bimanual（双臂+新动作空间）
    • 统一配方: ~100 条示范，<5h 单 GPU
    • 结果: Octo 平均 72% vs ResNet+Transformer Scratch 20% vs VC-1 15%（平均领先 52%）
    • 关键: 同一套超参数在所有场景通用

    C. 设计决策消融¶

    设计选择 成功率
    Octo-Small (完整) 83%
    换 RT-X 数据子集 (350K) 60%
    仅 Bridge 单机器人数据 43%
    离散化动作 (cross-entropy) 18%
    连续动作 MSE 35%
    ResNet-50 + Transformer 70%

    关键发现: 1. 数据多样性最重要：25 数据集 >> 11 数据集 >> 单机器人数据 2. Diffusion head 大幅优于替代方案：diffusion >> MSE（hedging）>> discrete（精度差） 3. ViT 优于 ResNet：transformer-first 架构在大数据集上更好（但小数据集上 ResNet 更好） 4. 模型规模正相关：Octo-Base > Octo-Small > Octo-Tiny，更大模型更鲁棒

    D. 实验发现（附录 E，重要经验）¶

    有效的: - 观测历史（1帧历史有帮助，更多帧边际收益递减） - Action chunking - 小 patch size (16×16 > 32×32) - 大 shuffle buffer (500k，对多数据集训练至关重要)

    无效的: - MSE/Discrete action head - 预训练 ImageNet ResNet 编码器（无提升） - 本体感知输入（可能因果混淆） - 微调语言模型（冻结 t5-base 最好） - 相对 gripper 动作表示（减少重试行为）

    评价¶

    优势¶

    1. 首个完全开源 GRP：模型、代码、数据加载器、微调脚本全部公开，对社区贡献巨大，直接催生了大量后续研究
    2. 灵活的模块化设计：block-wise attention + readout token 允许微调时自由添加/移除输入输出，这在先前工作（RT-1, ViNT）中不可能
    3. 系统性消融：对架构、数据、训练目标、模型规模的全面消融为后续工作提供了宝贵的设计指导
    4. 统一微调配方：同一套超参数在 9 个不同场景通用，降低了使用门槛
    5. 数据规模：800k 轨迹，当时最大的跨具身操作数据集训练

    局限¶

    1. 无 VLM 预训练：transformer backbone 从头训练，缺乏互联网级语义知识（相比 π₀ 用 PaliGemma，语言理解和视觉泛化能力有限）
    2. 语言条件表现较弱：仅 56% 预训练数据有语言标注，语言条件性能明显弱于目标图像条件
    3. 腕部相机利用不足：仅 27% 数据含腕部相机，微调时有时第三人称单相机反而更好
    4. 模型规模较小：93M 参数，相比 RT-2-X (55B) 或 π₀ (3.3B) 小很多
    5. 仅支持 delta end-effector 控制预训练：筛选数据时排除了非 delta EE 控制的数据集，限制了预训练数据量
    6. 本体感知信息未能有效利用：实验发现加入本体感知反而降低性能（因果混淆问题）

    对 VLA 领域的贡献¶

    1. 开源生态奠基：Octo 是社区中使用最广泛的开源通用机器人策略之一，直接启发了 OpenVLA 等后续开源工作
    2. 验证了 cross-embodiment 预训练 + 微调范式：首次系统性证明在大规模跨机器人数据上预训练后微调到新域（包括新传感器/动作空间）的有效性
    3. Diffusion head 在 GRP 中的价值：首次在跨具身通用策略中验证 diffusion head 显著优于 MSE 和 discrete head，为 π₀ 等后续工作使用 flow matching/diffusion 提供了实验依据
    4. Transformer-first 架构：验证了在大数据集上 ViT 风格 "参数集中在 transformer" 优于 "ResNet 重 + transformer 轻" 的设计
    5. 实用的工程指南：附录中的"what worked / didn't work"对后续研究者极具参考价值

    与其他工作的关系¶

    • π₀ 的前身：Octo 团队的多位核心成员（Karl Pertsch, Kevin Black, Sergey Levine 等）后来参与了 π₀ 的开发。π₀ 可以看作是 Octo 的"VLM 升级版"——用预训练 VLM (PaliGemma) 替代从头训练的 transformer，用 flow matching 替代 DDPM diffusion，规模从 93M 提升到 3.3B
    • RT-1-X / RT-2-X 对比：Octo 用 93M 参数达到了 55B RT-2-X 的相当性能，但 Octo 的优势在于灵活的微调能力而非零样本性能
    • OpenVLA 对比：OpenVLA 走的是 VLM + discrete action token 路线（7B），而 Octo 走的是 transformer + diffusion head 路线（93M）。π₀ 论文中显示 Octo 在灵巧任务上弱于 π₀ 但优于 OpenVLA
    • Diffusion Policy 对比：Octo 将 Diffusion Policy 的思想（diffusion head + action chunking）扩展到了跨具身大规模预训练场景
