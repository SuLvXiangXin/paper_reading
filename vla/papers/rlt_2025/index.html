    RLT: RL Token — Bootstrapping Online RL with Vision-Language-Action Models¶

    基本信息¶

    • 作者: Charles Xu, Jost Tobias Springenberg, Michael Equi, Ali Amin, Adnan Esmail, Sergey Levine, Liyiming Ke
    • 机构: Physical Intelligence
    • 时间: 2025
    • 链接: https://pi.website/research/rlt
    • 基座模型: π0.6（SigLIP 400M + Gemma 4B VLM + 860M Action Expert）
    • 方法线归属: VLA RL 后训练 (RL Post-training)

    一句话总结¶

    在冻结 VLA 上训练 encoder-decoder transformer 暴露紧凑 "RL Token" 表征，作为轻量 actor-critic 的状态接口，实现数小时真机在线 RL 微调，关键阶段速度提升 3×，部分任务超越人类遥操作速度。

    核心问题¶

    VLA 预训练策略在"最后一毫米"精细操作上表现不佳（慢、试探、失败），需要在线 RL 改善，但： 1. 全模型 RL 微调太慢：十亿参数模型的在线 RL 计算和样本效率都不够 2. 从头训练小模型丢失泛化：SERL 等方法虽快但丢弃了 VLA 的预训练先验 3. 核心矛盾：如何同时利用 VLA 的泛化能力和轻量在线 RL 的速度

    方法¶

    Step 1: 训练 RL Token（离线阶段）¶

    • 在冻结 VLA 的内部表征 z₁:M 上添加一个 encoder-decoder transformer
    • Encoder: 将 VLA 所有 token embeddings + 一个可学习 RL token embedding erl 压缩为 zrl（1×2048 向量）
    • Decoder: 从 zrl 自回归重建原始 VLA embeddings（重建损失 Lro 确保信息保留）
    • 在少量任务特定示教数据上训练（可选同时微调 VLA 权重）
    • 训练完毕后 冻结 VLA + RL token 模块

    Step 2: 在线 RL 微调（数小时真机）¶

    • Actor πθ(a₁:C | x, ã₁:C): 轻量 MLP，输入 = RL token zrl + 本体感知 sp + VLA 参考动作 ã
    • 输出高斯分布的 action chunk（C=10 步，14维/步 → 140维）
    • 条件化于 VLA 参考动作 → 局部精修而非从头搜索
    • Critic Qψ(x, a₁:C): 轻量 MLP，chunk-level Q 值估计
    • Off-policy TD 学习: 高 update-to-data ratio = 5，TD3 风格
    • BC 正则化: β‖a - ã‖² 锚定到 VLA 参考动作，防止探索漂移
    • 参考动作 dropout: 随机将参考动作置零，防止 actor 简单复制 VLA 输出

    关键设计选择¶

    1. Action Chunking RL（C=10）: 缩短有效决策时域，解决稀疏奖励下的信用分配问题
    2. VLA 参考动作传递: 保留 VLA 多模态分布的 mode 信息（单模态高斯无法恢复）
    3. 分阶段精修: RL 只改善任务的 关键阶段（插入/旋紧/穿线），简单阶段由 VLA 完成
    4. 异步训练: rollout 和学习异步执行，critic 更新 2 次 / actor 更新 1 次
    5. Subsampling: stride=2 存储中间步到 replay buffer，增加数据效率
    6. 人类干预: 可选的遥操作干预，数据也存入 replay buffer（off-policy 可复用）

    自主执行（test-time）¶

    • 训练结束后短暂微调 VLA 预测"何时切换到 RL 策略"（用人类切换时间点作标签）
    • 测试时自动触发 VLA → RL 策略切换

    实验结果¶

    四个真机任务（50Hz, 14维动作空间）¶

    任务 难度 关键阶段 基线成功率 → RLT 速度提升
    螺丝安装 极高（亚毫米对准） 螺丝插入+旋紧 20% → 65%（全任务） ~2×
    扎带穿线 高（双臂协调+柔性体） 尾端穿过锁扣 低 → +60%（全任务） ~2×
    网线插入 中（角度+力） 连接器对准+插入 高（维持） 3×（超人类遥操作）
    充电器插入 中（对准+可观测性差） 插头对准+插入 高（维持） ~3×

    vs 基线对比（Ethernet 任务）¶

    方法 成功率 吞吐量 说明
    Base VLA (π0.6) 高 低 探试行为，速度慢
    DAgger ~RLT 低 受限于人类示教速度
    HIL-SERL 差 差 单步 RL，无法处理长时域稀疏奖励
    PLD 差 差 单步残差，同上
    DSRL ~RLT 中 噪声空间 RL，强约束限制改进幅度
    RLT (Ours) ~最优 最高 2× 步数减少

    消融实验（Ethernet）¶

    • 去掉 RL Token → 吞吐量降 50%（ResNet-10 编码器不含操作相关结构）
    • 去掉 Chunk（C=1） → 无法匹配基线（有效时域过长，信用分配失败）
    • 去掉 BC 正则化 → 最大单项下降（无约束探索导致崩溃）
    • 去掉参考动作传递 → 学习变慢，早期探索漂移，最终可追上

    涌现行为¶

    • 网线插入: 基线 VLA 反复"探试"（接近→退回→再试），RLT 学会流畅一次插入
    • 失败时学会利用柔顺性"摇摆"连接器 → 示教数据中未出现的策略
    • 中位完成步数: 遥操作 146 → 基线 VLA 228 → RLT 66（人类的 45%）

    训练效率¶

    • RL Token 训练: 少量任务示教数据（1-10 小时遥操作数据）
    • 在线 RL: 400-1000 episodes，实际机器人数据约 15 分钟 ~ 5 小时
    • 5 分钟数据即超越基线（Ethernet 关键阶段）
    • 人类监督: 稀疏二值奖励（成功/失败标注）+ 可选干预

    与现有方法的关系¶

    vs RECAP/π*0.6（全模型 RL）¶

    • RECAP 端到端更新整个 π0.6（离线 RL + advantage-conditioned），计算量大
    • RLT 冻结 VLA，只训练轻量 actor-critic → 分钟级改善 vs 大规模离线迭代

    vs Hi-ORS（拒绝采样 RL 后训练）¶

    • Hi-ORS 用 rejection sampling 替代 Q 函数，完全 value-free
    • RLT 用标准 TD 学习 + Q 函数，但限制在轻量 actor-critic 上 → 可以发现新策略（超人类速度）
    • Hi-ORS 更保守（基于采样过滤），RLT 更激进（可探索新行为）

    vs Evo-RL（离线 ACP）¶

    • Evo-RL 完全离线，优势标签注入 task text，无需在线探索
    • RLT 需要在线交互但可发现示教数据中不存在的策略

    vs DSRL/GR-RL（噪声空间 RL）¶

    • 在 diffusion/flow 的噪声空间学习，隐式约束在 VLA 可达动作内
    • RLT 在动作空间直接操作，BC 正则化 + 参考动作提供更灵活的约束
    • 速度提升显著优于 DSRL（RLT 吞吐量高得多）

    vs ConRFT（冻结 VLA + 微调 action head）¶

    • ConRFT 单步动作，无 chunking，短时域任务
    • RLT chunk-level RL，解决长时域稀疏奖励问题

    技术亮点¶

    1. RL Token 瓶颈设计: encoder-decoder 重建迫使 zrl 保留所有任务相关信息的精华
    2. 分而治之: VLA 负责简单阶段 + 提供表征/参考动作，RL 只精修关键阶段
    3. 超人类速度: 通过 RL 发现了人类遥操作者未展示的更高效策略
    4. 实用性: 现实约束（稀疏奖励、有限交互时间、人类标注）下可工作

    局限性¶

    • 训练期间需人类提供奖励信号、干预修正、策略切换时机
    • 每个任务需独立训练 RL（非通用策略）
    • 仅评估关键阶段改善，非端到端自主改善
    • 未来方向: 奖励模型 + 进度预测 → 全自主 RL 改善流水线
