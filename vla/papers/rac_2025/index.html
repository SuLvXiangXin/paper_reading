    RaC: Robot Learning for Long-Horizon Tasks by Scaling Recovery and Correction (2025)¶

    基本信息¶

    • 作者: Zheyuan Hu, Robyn Wu, Naveen Enock, Jasmine Li, Riya Kadakia, Zackory Erickson, Aviral Kumar
    • 机构: Carnegie Mellon University
    • arXiv: 2509.07953
    • 发布日期: 2025-09-09
    • 项目页: https://rac-scaling-robot.github.io/

    一句话总结¶

    在模仿学习预训练后，通过人在回路数据收集显式扩展"恢复+修正"行为，用 10x 更少数据在双臂长时序操作任务（衬衫挂架 78.3%）超越 SOTA，并首次证明机器人策略的 o1-style 测试时 scaling：恢复次数越多成功率线性增长。

    问题¶

    模仿学习（IL）在长时序任务上遭遇性能天花板：即使有超过 5000 条专家演示，双臂衬衫挂架任务成功率也仅约 75%（ALOHA Unleashed，89小时数据）。根本原因不是模型或算法，而是数据分布：

    • 专家演示仅覆盖"干净成功"轨迹，缺乏失败恢复行为
    • 长时序中误差累积（compounding errors），单步失败导致后续全失败
    • 分析 DROID 数据集发现：仅 3.68% 的 episodes 包含恢复行为

    核心问题：如何设计数据收集策略，使策略能在实际部署中的随机扰动和分布外状态下保持鲁棒？

    方法¶

    • 方法线归属: 交互式数据收集 / Human-in-the-Loop 模仿学习
    • 核心 idea: 在 IL 预训练后，增加一个专门收集"恢复（Recovery）+ 修正（Correction）"行为的人在回路数据收集阶段。与 HG-DAgger 的关键区别：Recovery 不是最优行为，但比 Correction 更容易学习——将机器人回退到熟悉的分布内状态，然后给策略第二次机会重试。

    核心技术：两条规则¶

    Rule 1: Recover-then-Correct 配对¶

    每次人类介入必须分两阶段： 1. 恢复段（Recovery）：人类将机器人引导回到历史演示中出现过的"熟悉"状态（分布内） 2. 修正段（Correction）：人类示范完成当前子任务

    这与 HG-DAgger 的纯修正形成对比。分析显示： - HG-DAgger 数据 recovery:correction 比约 1:10（严重偏向修正） - RaC 数据维持约 1:2 的均衡比例（27%~45% 恢复帧）

    Rule 2: 介入后立即终止¶

    人类介入结束后立即终止 episode（不让策略继续执行后续子任务）。

    原理：介入后的后续状态是人类+策略联合分布产生的，与纯策略推理时的状态分布不一致，继续收集会"污染"后续子任务的监督信号；及早终止将预算更多投入改善早期子任务。

    为何 Recovery 有效？¶

    "Verification-Generation Gap"（VG-gap）： - 长时序任务中，初始状态分布宽（衣服在桌上的任意位置），目标状态分布窄（挂架精确插入衣领） - 回退到历史分布内状态比前进完成子任务精度要求低得多 - 因此 recovery 技能比 correction 技能更容易学习（需要更少数据） - 一旦策略学会 recovery，多次重试使成功概率指数级增加：P(从不成功) = (1-p)^n → 0

    数据收集协议¶

    Round 0: 收集初始专家演示 D0 → 训练初始策略 π0
for k = 1 to K:
    部署 π(k-1)，人类监控
    若失败迹象 → 介入：
        [Recovery] 将机器人回退至熟悉状态
        [Correction] 完成当前子任务
        立即终止 episode（Rule 2）
    无介入的完整成功轨迹也加入数据集
    训练新策略 π_k

    介入界面：VR 共享自主（Shared Autonomy）¶

    • 使用 Oculus Quest VR 控制器（off-the-shelf，无需专用设备）
    • "离合器"机制（Clutch）：
    • 按住侧键 → VR 控制器动作直接映射到末端执行器（人类接管）
    • 松开侧键 → 机器人执行学习策略（策略接管）
    • 无需对齐 VR 姿态与机器人末端执行器姿态（本地帧注册 + 相对位移控制）
    • 视觉辅助：用 SAM2 追踪历史演示中夹爪轨迹，叠加频率热图，帮助操作者找到合适的恢复目标位置

    策略架构（Policy Architecture）¶

    • 模型：368M 参数多模态扩散 Transformer（MM-DiT）
    • 训练目标：Flow Matching（条件流匹配）
    • 视觉编码器：ResNet-50（3路：1个俯视 + 2个腕部摄像头）
    • 动作 chunk：H=60 步（~1秒），每 0.5 秒重新规划
    • 推理：10步 Euler 积分从噪声生成动作

    实验¶

    任务（均为双臂双 7-DoF xArm-7）¶

    1. shirt-hanging（5 子任务）：取挂架→换手→左右衣领插入→复位
    2. airtight-container-lid-sealing（5 子任务）：取盖→放置→扣两个卡扣→旋转容器→扣剩余卡扣
    3. clamshell-takeout-box-packing（7 子任务）：取盒→取铲→铲汉堡→装盒→调位→合盖→锁扣
    4. bimanual-assembly（仿真，3 子任务）：插白块→套粉槽→放平台

    主要结果¶

    方法 数据量 衬衫挂架 SR
    ALOHA Unleashed ~89小时（5345 demos） 75.0%
    Seed GR-3 (4B VLA) 116小时 + VL 数据 ~63.6%
    RaC（本文） 5小时（恢复+修正数据） 78.3%

    数据效率：与 batched full demo 相比，同等预算下性能高 2x 以上；与 HG-DAgger 相比，始终更优且 scaling 斜率更陡（每单位数据获益更多）。

    Test-time Scaling（类 o1 的测试时 Scaling）¶

    关键发现：成功轨迹中恢复行为的平均次数与成功率呈线性相关： - Shirt hanging: r=0.714, y = 1.169x - 0.387 - Airtight lid: r=0.803, y = 0.179x + 0.099 - Takeout box: r=0.877, y = 0.253x + 0.111

    类比 LLM：长 CoT + 回溯 → 更好推理性能；Robot RaC：更多 recovery segments → 更高成功率。RaC 策略的成功轨迹更长（中位数 53s vs HG-DAgger 48s vs Full Demo 43s），但成功率最高（61% vs 22% vs 23%）。

    消融实验¶

    • 去掉 Rule 1（不强制 recovery）：recovery/correction 比例退化至 1:6~1:7，性能下降
    • 去掉 Rule 2（不终止）：与 Rule 1 去掉叠加，等同于 HG-DAgger，scaling 更差
    • 两条规则缺一不可，Rule 2 的独立贡献也显著

    评价¶

    优势¶

    1. 洞察深刻：识别出数据分布偏向"成功轨迹"是 IL 性能瓶颈的根本，而非模型能力
    2. VG-gap 理论：恢复比修正更容易学这一 insight 有理论依据，类比 LLM 中 generation vs verification 差异
    3. 极高数据效率：5小时数据超越 89小时数据，10x 数据效率提升有说服力
    4. Test-time scaling 的机器人类比：首次在真机上证明机器人策略的 o1-style scaling，线性相关系数高
    5. 实用性强：只改数据收集策略，不改 IL 目标，可插拔到任何 IL 系统

    局限¶

    1. 依赖 VR 硬件：Oculus Quest VR 控制器，虽比 VR exoskeleton 便宜，仍不如 HACTS 等低成本方案（<$300）
    2. 不能并行部署多机器人：每次 rollout 需要人类持续监控，扩展性有限
    3. 恢复段设计仍依赖人类判断：何时恢复、恢复到哪个状态由操作者判断，标准化程度有限
    4. 评测任务数量少（3真机+1仿真），泛化性待验证
    5. 只适用于长时序多子任务：对短时序单任务提升有限（recovery 的作用不突出）

    与相关工作对比¶

    方法 核心机制 Recovery? 数据效率
    HG-DAgger 人类修正（仅 correction） 少量偶发 中
    RoboCopilot 顺应性双臂遥操作 + correction 无显式 中
    GCENT/Genie Centurion 倒带+Task Sentinel自动监控 有（自动） 高（Task Sentinel）
    RaC（本文） 恢复+修正显式配对 + 两条规则 显式强制 最高（10x+）

    与 GCENT 的关系¶

    两者都引入了"倒带/恢复"机制： - GCENT：用 Task Sentinel 自动检测失败，自动回到历史帧的状态，侧重单人监控多机器人 - RaC：人类主动执行恢复动作，侧重数据构成理论和 test-time scaling；RaC 更强调 recovery skill 的可学性 vs correction skill 的 VG-gap

    对知识库的贡献¶

    • 数据收集方法线：RaC 填补了"显式 recovery 数据 + 两条规则标准化"的空缺
    • Test-time Scaling：首次在真机机器人策略中证明 recovery 次数与成功率的线性 scaling
    • VG-gap 概念：recovery 比 correction 更容易学的理论解释，可应用于分析其他 HITL 方法的数据构成
