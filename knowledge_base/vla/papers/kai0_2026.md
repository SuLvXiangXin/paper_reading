# χ0: Resource-Aware Robust Manipulation via Taming Distributional Inconsistencies (2026)

## 一句话总结
χ0 / kai0 不是单纯扩大数据或模型，而是把长时序衣物操作中的失败归因到 demonstration、policy bias 和 deployment trajectory 之间的分布不一致，并用 Model Arithmetic、Stage Advantage、Train-Deploy Alignment 在有限资源下把 π0.5 提升到生产级双机器人协作鲁棒性。

## 问题
衣物 flattening、folding、hanging 这类长时序双臂任务会放大三个分布问题：专家数据覆盖不足、不同阶段视觉相似导致策略误用、部署延迟和失败状态导致误差级联。仅增加数据或直接 fine-tune VLA 成本高，而且不一定覆盖失败恢复。

## 方法
- 方法线归属: VLA 后训练 / 分布对齐 / 长时序双臂操作
- 核心 idea: 显式对齐 Ptrain、Qmodel、Ptest 三个分布，而不是把鲁棒性完全交给规模化数据。
- 关键技术点:
  - Model Arithmetic: 将在互补数据子集上 fine-tune 的多个策略在权重空间合并，用 OOD/DAgger validation loss 选择 merge 权重，提高覆盖而不新增大量采集。
  - Stage Advantage: 直接预测阶段条件下的相对进展 A(s, a, g)，避免用两个 value 差分造成高方差，并解决长任务阶段歧义。
  - Train-Deploy Alignment: 用 spatio-temporal augmentation、heuristic DAgger failure-state correction 和 temporal chunk-wise smoothing 减少训练-部署差距。
  - 系统目标: 两套双臂 ALOHA 机器人协作完成 flattening、folding、handover、hanging 等衣物流程。

## 实验
- Benchmark: 三个真实衣物任务：T-shirt Flattening and Folding、Conditional Retrieval and Sorting、Garment Hanging；每任务约 20 小时专家演示，10 trials × 3 garment types。
- 主要结果: 完整 χ0 的模块叠加带来单调提升；论文报告相对 π0.5 baseline 约 250% 成功率提升，并可从任意初始状态连续运行 24 小时。
- 对比基线: π0.5、π0，以及 GO-1、X-VLA、DexVLA 等尝试性基线；组件消融包括 MA、SA、TDA、DAgger 与不同控制平滑策略。

## 评价
- 优势: 关注真实部署中最常见的分布错配，尤其是长时序、可变形物体、延迟和恢复；工程上比“继续堆数据/模型”更资源友好。
- 局限: stage label、heuristic failure state 和系统调参仍有人工设计成分；任务集中在衣物操作，方法的通用性需更多平台验证。
- 对 VLA 领域的贡献: χ0 将 VLA 后训练从模型架构问题推进到部署分布对齐问题，是 π0.5 系策略走向可靠长时序生产任务的重要工程化分支。
