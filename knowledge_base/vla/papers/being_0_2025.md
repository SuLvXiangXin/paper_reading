# Being-0: A Humanoid Robotic Agent with Vision-Language Models and Modular Skills (2025)

## 基本信息
- 作者: Haoqi Yuan, Yu Bai, Yuhui Fu, Bohan Zhou, Yicheng Feng, Xinrun Xu, Yi Zhan, Borje F. Karlsson, Zongqing Lu
- 机构: Peking University, BAAI, BeingBeyond
- arXiv: 2503.12533
- 项目: https://research.beingbeyond.com/being-0

## 一句话总结
Being-0 是分离式 humanoid agent 系统：云端 Foundation Model 负责语言理解、任务分解和高层推理；本地轻量 VLM Connector 把语言计划 grounded 成可执行技能命令；Modular Skill Library 用 RL locomotion + ACT manipulation 技能完成低层控制。

## 问题
直接让 GPT-4o 这类 FM 观察图像并调用 humanoid 技能，会遇到三类问题：长时序任务中错误会累积，FM 对机器人相对位置/深度/可操作性的 embodied scene understanding 不稳定，且云端推理慢，无法以 humanoid 双足导航所需频率做闭环修正。Being-0 的目标不是训练端到端 VLA，而是把慢速高层认知和快速低层技能之间的接口做实。

## 方法
- 方法线归属: VLM-as-Controller / hierarchical humanoid agent；模块化 skill policy；不是单一 VLA action head。
- 核心 idea: 用 FM 做低频 planner，用 onboard VLM Connector 做中频 grounded skill planner/navigation coordinator，用预训练技能做高频控制；把“语言计划 -> 技能调用 -> 关节控制”拆成可部署的层次结构。
- 关键技术点:
  - **三层架构**:
    - FM: 基于 GPT-4o/Cradle 风格 agent prompt，做图像描述、成功检测、自反思、子任务推理和下一技能选择。
    - Connector: 基于 VideoLLaMA2/VideoOrion 风格轻量 VLM，部署在 onboard 设备，约 1s 推理；输入 FM 的语言计划/子任务 + 当前第一视角图像，输出 grounded skill command。
    - Skill Library: locomotion skill 控制下半身，manipulation skill 控制上肢/灵巧手，最终输出 PD target joint positions。
  - **Modular skills**:
    - Locomotion 由仿真 RL 训练，50Hz 部署，技能包括 no_action、go_straight、walk_backwards、turn_left/right、sidestep_left/right、tilt_head、turn_head。
    - Manipulation 用 Apple VisionPro 遥操作采集，再用 ACT/ResNet-50 行为克隆训练；每个技能几十到数百条轨迹，训练 chunk=30，部署 chunk=10，10Hz 控制。
  - **VLM 规划到低层技能接口**:
    - Visual understanding 输出目标 bounding box/None、yes/no、图像描述等结构化信号。
    - Skill planning 在 overall task + subtask 条件下输出技能库中的函数式 skill code，或 move_towards/search_for/adjust 这类 composite navigation skill。
    - Navigation 用 VLM 检测框 + binocular synthetic depth 估计目标相对位置，再选择前进/转向/侧移；找不到目标时触发 search；到达目标前用 adjustment 沿弧形路径调整站位，使后续 manipulation 初始姿态更可靠。
  - **与 VLA 的关系**: Being-0 更接近 SayCan/Inner Monologue/RoboClaw 式系统框架，而非 π0/OpenVLA 式端到端 VLA。它的低层技能可以未来替换成 VLA policy，但本文接口粒度是 symbolic/composite skills，不是连续 action chunk。

## 实验
- Benchmark: Unitree H1-2 + Inspire hands + active neck camera，在约 20m x 20m 办公环境评测导航、操作和长时序任务。
- 主要结果:
  - 长时序任务完成率：Fetch-bottle 0.90、Deliver-basket 0.80、Prepare-coffee 0.75、Make-coffee 0.90、Deliver-coffee 0.87，平均约 84.4%。
  - w/o Connector 对应为 0.00、0.00、0.00、0.90、0.33，说明 Connector 对涉及导航+操作切换的任务最关键。
  - Navigation efficiency: Being-0 9.6 cm/s 且 5/5 成功；w/o Connector 2.3 cm/s 且 0/5。
  - Active camera ablation: 固定相机在导航和桌面操作间存在 pitch trade-off；active camera 在 table/coffee-machine navigation 和 grasp/place coffee 上均 5/5。
  - Manipulation robustness: seen object/perturbation 下 grasp-bottle、handout-snack、place-pole、play-chess 多数约 0.77-0.90；unseen object 有下降但仍可执行。
- 对比基线: 主要是 w/o Connector、w/o pose adjustment、fixed camera；不是与端到端 VLA benchmark 横向比较。

## 评价
- 优势: 清楚展示 humanoid agent 的系统接口：FM 不直接控制机器人，而是低频生成任务语义；Connector 负责把语义计划实时 grounded 到技能库；技能 policy 保持高频稳定控制。对人形机器人，这比直接端到端训练一个大 VLA 更容易工程落地。
- 局限: 技能库是人工定义且每个 manipulation skill 需要遥操作数据；系统依赖云端 FM，开放世界扩展会受延迟和 prompt 可靠性影响；实验集中在平地办公场景，不包含蹲下、坐下、跳跃、楼梯等复杂 whole-body locomotion。
- 对 VLA 领域的贡献: Being-0 是“VLA-as-skill 或 skill-as-action-space”系统路线的重要参照。它说明 humanoid 长时序任务的瓶颈不仅是低层动作生成，还包括 planner 与低层控制之间的 grounded interface；未来端到端 VLA 若要上 humanoid，也需要类似 Connector 的空间 grounding、技能切换、站位调整和成功检测能力。
