# Being-H0.5: Scaling Human-Centric Robot Learning for Cross-Embodiment Generalization (2026)

## 基本信息
- 作者: BeingBeyond Team
- 机构: BeingBeyond
- arXiv: 2601.12993
- 链接: https://research.beingbeyond.com/being-h05
- 日期: 2026.01.19

## 一句话总结
以人手运动为"物理母语"，构建迄今最大规模的具身预训练语料 UniHand-2.0（35K+ 小时、30 种具身、120B token），通过统一动作空间 + Mixture-of-Flow 架构 + 流形保持门控等技术，实现单一 VLA 检查点在 5 种异构机器人平台上的跨具身泛化控制，LIBERO 98.9%、RoboCasa 53.9% 刷新 SOTA。

## 问题
当前 VLA 模型本质上是"单语者"——在特定硬件上高度熟练，但换一种形态就失效。核心困难有三：
1. **数据稀缺**：大多数机器人平台缺乏大规模示范数据，尤其灵巧手数据不到现有语料的 5%
2. **具身异构性**：人手、夹爪、灵巧手、人形机器人的运动学/动作空间截然不同，朴素聚合数据会导致负迁移
3. **流形漂移**：对 diffusion/flow-based VLA 而言，在简单具身（夹爪）上预训练的速度场应用于高 DoF 具身时，因分布偏移导致轨迹"漂出"有效运动流形，产生不稳定行为

**核心问题**：如何像人类学习"小语种"那样，让 VLA 仅用少量目标数据即可在全新机器人形态上运行？

## 方法
- **方法线归属**: VLM + Diffusion/Flow Head（继承 π₀ 系列），加入 **人类中心学习** + **跨具身统一动作空间** + **MoT/MoF 架构创新**
- **核心 idea**: 将人手运动视为所有物理交互的"通用母语"，构建统一动作空间将人手和 30 种机器人映射到语义对齐的槽位，通过 Mixture-of-Flow 架构解耦共享运动原语与具身特有专家，辅以流形保持门控和通用异步分块实现稳定的跨具身部署。

### 关键技术点:

#### 1. UniHand-2.0 数据集（核心数据贡献）
迄今最大的具身 VLA 预训练语料：
| 数据源 | 规模 | 描述 |
|--------|------|------|
| 人类示范 | 16K 小时（45.7B token） | 从 Ego4D/EPIC-KITCHENS 等提取 MANO 手部运动，Gemini-2.5 生成双层语义标注 |
| 机器人操作 | 14K 小时（50.2B token） | 30 种具身（OXE/AgiBot/SO100 等），仿真占 26%，帧率降采样 30% |
| 视觉文本理解 | 5K 等效小时（25.6B token） | VQA + 2D 空间定位 + 任务规划推理，平衡视觉-文本模态比 |
| **总计** | **35K+ 小时、400M 样本、120B token** | 相比 UniHand-1.0 扩大 200× |

- 30 种具身（以 >10 小时为计）是 VLA 文献中最多的（π₀.5 约 10 种）
- 人类数据经过 4 阶段后处理：语言增强、运动质量过滤、操作相关性过滤、左右手去偏

#### 2. UniCraftor 数据采集系统
自研模块化采集系统，弥补现有数据集缺乏深度/外参/精确事件时间戳的不足：
- RealSense D435 原生深度采集
- AprilTag PnP 高精度外参
- 脚踏板硬件同步交互事件标注
- 已采集 200+ 小时、43 个桌面任务

#### 3. 统一动作空间（Unified State-Action Space）
- 将人手 MANO 参数和异构机器人控制映射到**固定长度高维向量**，按物理语义分隔子空间：EEF 位姿、夹爪状态、手指关节、底座速度等
- **关键创新**：人手腕部位姿 → EEF 子空间，手指关节 → "精细操作"保留槽位
- 统一旋转为轴角表示（避免万向锁），保留原始物理量级（不做统计归一化）
- 与 π₀.5 的对比：π₀.5 用独立 MLP 头适配不同具身，Being-H0.5 用统一空间 + 稀疏槽位激活，理论上更利于跨具身迁移

#### 4. Mixture-of-Transformers (MoT) + Mixture-of-Flow (MoF) 架构
- **MoT 主干**：理解专家（文本/图像，共享注意力 + 独立 FFN）和动作生成专家（flow matching，共享注意力 + 独立 FFN），继承 BAGEL/Transfusion 风格
- **MoF 动作模块**：
  - **基础专家（底层）**：共享 transformer 块，编码通用运动原语（抓取、碰撞回避等）
  - **特化专家（顶层）**：Top-K 路由的并行专家模块，按具身/任务动态激活
  - 灵感来源于 MoE 架构，实现参数量与激活参数量解耦，支持边缘设备部署
- **VLM backbone**：InternVL-3.5（decoder-only）

#### 5. 混合人类运动表示（Hybrid Human Motion Representation）
- 同一训练实例中同时监督**连续动作块**（flow matching）和**离散运动 token**（masked prediction）
- 连续通道：Rectified Flow matching，预测速度场 v_θ(x_t, t, c)
- 离散通道：预训练 tokenizer 将运动量化为 codebook 索引，masked token prediction（类 BERT）
- 联合序列化：两个目标通过注意力掩码互相不可见，但共享同一上下文前缀，位置编码从同一起点对齐
- **与 π₀.5 的异同**：π₀.5 也用 FAST discrete token + flow matching 混合，但 Being-H0.5 的 masked prediction 用于人手运动先验（稳定行为抽象），而 π₀.5 的 FAST token 主要用于预训练效率

#### 6. 流形保持门控（Manifold-Preserving Gating, MPG）
- 解决问题：分布偏移下（光照变化、视角扰动、遮挡），上下文特征 H 退化导致 flow matching 去噪产生动作抖动
- 方法：用 Sliced-Wasserstein Distance 衡量观测嵌入与无噪声动作锚点的分布差异，生成 temperature-scaled gate g ∈ (0,1]
- 当 g≈0（不可靠），只保留无门控的学习先验偏置 b_MPG；当 g≈1（可靠），允许特征依赖修正
- 相比 DiG-Flow 的输出门控，MPG 将门控应用于投影前，方差更低、轨迹更平滑
- stop-gradient 防止模型操纵 gate 而非改善底层特征

#### 7. 通用异步分块（Universal Async Chunking, UAC）
- 解决问题：不同机器人控制频率和推理延迟不同，统一 action chunking 策略无法跨平台
- 训练时为每条轨迹随机采样 latency offset d ~ D_e（具身特定延迟分布），将动作块分为已提交前缀 A_{<d} 和可预测后缀 A_{≥d}
- 推理时：双线程架构（推理线程 + 执行线程），通过锁和 FIFO 异步协调
- 支持五种异构平台的统一部署，一个检查点适配不同延迟

#### 8. 具身特定适配（Embodiment-Specific Adaptation, ESA）
- 在统一动作空间中维护 slot-wise adapter bank，每种具身只更新其激活槽位对应的 adapter
- 共享硬件组件（如同一手臂+不同末端）共享适配参数子集
- 避免传统独立投影头的参数碎片化

### 训练流程
- **预训练**：UniHand-2.0 全量统一序列建模，L = λ_text L_text + λ_act L_act
- **后训练**：ESA + MPG + UAC，适配目标具身

## 实验

### 仿真 Benchmark
| Benchmark | 指标 | Being-H0.5 | 对比最优 |
|-----------|------|-----------|----------|
| LIBERO-Long | 成功率 | **98.9%** | π₀ 93.4%, GR00T-N1 93.4% |
| RoboCasa | 成功率 | **53.9%** | InternVLA-M1 49.9% |

- 仅用低分辨率 RGB，无辅助模态

### 真机跨具身评测（5 种平台）
| 平台 | 类型 | 任务数 | Being-H0.5 (Specialist) | π₀.5 (Specialist) | Being-H0.5 (Cross-Emb) |
|------|------|--------|------------------------|-------------------|----------------------|
| PND Adam-U | 灵巧手半人形 | 5 | **75.5%** | 52.0% | 70.0% |
| Franka+Inspire | 灵巧手臂 | 5 | **73.0%** | 52.0% | 68.0% |
| Unitree G1 | 人形机器人 | 5 | **64.5%** | 44.0% | 57.5% |
| BeingBeyond D1 | 教育臂 | 5 | **57.5%** | 36.0% | 50.0% |
| SO-101 | 便携臂 | 5 | **64.0%** | 44.0% | 59.0% |

- 跨具身单一检查点在多数平台上接近甚至匹配专用检查点
- **涌现发现**：单一跨具身检查点在**从未见过的任务-具身对**上展现非零成功率（zero-shot transfer），如 Franka+Inspire 在仅训练于其他平台的 wipe whiteboard 任务上达 15% 成功率

### 四维能力分解（Space / Long-Horizon / Bimanual / Generalization）
- Being-H0.5 在所有四个维度上均大幅超越 π₀.5 和无预训练基线
- 泛化维度提升最显著：Being-H0.5 75-80% vs π₀.5 50-60%

### 消融实验
1. **人类中心学习的贡献**：预训练显著提升下游适应（vs 无预训练基线）；人类数据提供的行为先验在灵巧手任务中尤其关键
2. **Masked Motion Token Prediction**：提升行为先验捕获，提高运动连贯性
3. **MPG + UAC 实时效率**：MPG 降低分布偏移下的动作抖动；UAC 使跨平台推理延迟一致

## 评价

### 优势
1. **数据规模与多样性无出其右**：UniHand-2.0 的 35K 小时 / 30 种具身 / 120B token 是迄今最大的具身 VLA 预训练语料，相比 π₀.5 的 ~10 种具身、Being-H0 的 175 小时人类数据均有量级提升
2. **统一动作空间设计深思熟虑**：将人手 MANO 参数和机器人控制映射到物理语义对齐的槽位，保留原始物理量级（不做归一化），比 π₀.5 的独立头方案更利于跨具身迁移
3. **MoF 架构优雅解耦**：共享运动原语 + 路由专家的设计同时解决了容量瓶颈和跨具身干扰，并且支持边缘部署（稀疏激活）
4. **MPG 是有理论支撑的工程创新**：用分布距离衡量上下文可靠性 → 门控 feature residual + 无门控先验偏置，比朴素门控更稳定
5. **完整的开源承诺**：模型、训练管线、部署基础设施、1000 GPU-hour 预训练配方全部开源，可复现性远超 π₀.5
6. **涌现 zero-shot 跨具身迁移**：单一检查点在未训练的任务-具身对上展现非零成功率，暗示统一动作空间确实学到了可迁移的物理先验

### 局限
1. **真机评测任务相对简单**：以桌面操作为主（stack bowls, wipe whiteboard 等），缺乏 π₀.5 级别的长时序多阶段家庭任务（10-15 分钟清洁厨房/卧室）
2. **无层次化推理**：不像 π₀.5 那样支持高层子任务分解 + 低层动作的统一层次化推理，限制了长视野任务能力
3. **统一动作空间的可扩展性未充分验证**：随具身类型增加，固定长度向量的槽位分配是否会成为瓶颈？
4. **人类数据的质量依赖**：依赖 HaWoR 进行手部姿态估计，在野外视频中的精度有限（论文自身提到需要多阶段过滤）
5. **仿真 Benchmark 过于饱和**：LIBERO 98.9% 已接近天花板，难以区分不同方法；RoboCasa 53.9% 虽 SOTA 但绝对值仍不高
6. **缺少与 EgoScale 的直接对比**：EgoScale 同样用 20K 小时 ego 视频做 flow-based VLA，两者方法论高度重叠，但论文未进行头对头比较

### 对 VLA 领域的贡献
1. **确立了"人类运动作为通用物理先验"的范式**：从 Being-H0 的概念验证到 Being-H0.5 的规模化实践，证明人手数据是跨具身泛化的有效"母语"
2. **统一动作空间为跨具身 VLA 提供了新范式**：不再为每种机器人维护独立头，而是通过物理语义对齐的共享空间实现知识迁移，是对 π₀.5 独立头方案的有力替代
3. **MoF 为 flow-based VLA 的容量扩展提供了架构模板**：共享基础层 + 路由专家层的设计可被后续工作广泛采纳
4. **MPG + UAC 为跨具身部署提供了实用工具**：解决了 distribution shift 和 latency mismatch 这两个工程痛点
5. **开源贡献推动社区发展**：在 π₀.5 闭源的背景下，Being-H0.5 的全面开源为社区提供了可复现的跨具身 VLA 基线

### 与 π₀/π₀.5 的关系
- **继承**：MoT 双专家设计（理解+动作，共享注意力）、flow matching 动作生成、预训练/后训练范式
- **差异化**：
  - π₀.5 用独立 MLP 头适配不同具身 → Being-H0.5 用统一动作空间 + 稀疏槽位
  - π₀.5 用 FAST discrete token 做预训练效率优化 → Being-H0.5 用 masked motion token prediction 做行为先验学习
  - π₀.5 有层次化推理（高层子任务 + 低层动作） → Being-H0.5 无此设计
  - π₀.5 主打开放世界长时序家庭任务 → Being-H0.5 主打跨具身泛化
  - π₀.5 闭源 → Being-H0.5 开源
- **互补**：π₀.5 在任务复杂度上更强，Being-H0.5 在具身多样性上更强

### 与 Being-H0 的关系
Being-H0.5 是 Being-H0 的直接后续：
- Being-H0 引入人手运动 tokenizer + UniHand-1.0（175 小时）→ Being-H0.5 扩展到 UniHand-2.0（35K 小时，200× 增长）
- Being-H0 仅做概念验证 → Being-H0.5 实现规模化跨具身泛化 + SOTA benchmark 性能
- Being-H0 的 motion tokenizer → Being-H0.5 的 hybrid 表示（连续 flow + 离散 masked prediction）
