# RoboClaw: An Agentic Framework for Scalable Long-Horizon Robotic Tasks (2026)

## 基本信息
- 作者: Ruiying Li, Yunlang Zhou, YuYao Zhu, Kylin Chen, Jingyuan Wang, Sukai Wang, Kongtao Hu, Minhui Yu, Bowen Jiang, Zhan Su, Jiayao Ma, Xin He, Yongjian Shen, Guanghui Ren, Maoqing Yao, Wenhao Wang, Yao Mu
- 机构: AgiBot + National University of Singapore + Shanghai Jiao Tong University (MoE Key Lab of AI)
- arXiv: 2603.11558
- 对应作者: Yao Mu (SJTU)

## 一句话总结
提出一个VLM驱动的系统级agentic框架，通过Entangled Action Pairs（正-逆动作对）实现自重置数据收集，并用VLM元控制器在数据收集和任务部署中共享同一决策循环，在长时序任务上比baseline提升25%成功率，同时减少53.7%人力投入。

## 问题
VLA系统在长时序任务中面临三重挑战：
1. **数据收集成本高**：每次操作后需要人工重置环境，大规模数据获取不可持续
2. **收集-训练-部署断裂**：三阶段由不同人/模块执行，导致任务语义不一致和训练-部署分布偏移
3. **长时序执行脆弱**：多策略串联执行时，缺乏运行时监督和恢复机制，错误易累积

核心矛盾：如何在数据收集、策略学习和任务执行之间建立统一的语义表示和决策机制？

## 方法
- **方法线归属**: **VLM-as-Controller 系统框架**（非新VLA模型架构）。使用现成VLM做高层推理+现成VLA(π₀.5)做低层控制，是一种分离式层次化方案，但强调全生命周期统一。
- **核心 idea**: 用一个VLM元控制器统一数据收集和任务执行的决策循环；通过正-逆动作配对（EAP）让机器人自动重置环境实现自主数据采集。

### 关键技术点:

#### 1. 系统架构：VLM元控制器 + 三层抽象
- 三层层次结构：**Skills**（可复用流程，如"long-horizon-execution"）→ **Tools**（MCP接口，如Start/Terminate/Switch Policy, Env Summary, Call Human）→ **Policies**（VLA模型，这里用π₀.5）
- VLM通过ICL（In-Context Learning）进行推理，输入为多模态观测 + 结构化记忆
- **结构化记忆 m_t = (r_t, g_t, w_t)**：
  - r_t：角色身份（当前运行模式 + 可用工具集）
  - g_t：任务级记忆（全局任务 + 子任务分解 + 执行状态）
  - w_t：工作记忆（当前激活skill + 工具调用历史）
- 决策循环：观测+记忆 → CoT推理 → 工具调用 → 结果写回记忆 → 循环

#### 2. Entangled Action Pairs (EAP) — 自重置数据收集
- 对每个操作策略k，训练一对策略：前向执行策略 π→_θk 和逆向恢复策略 π←_ϕk
- 前向策略执行任务（如"放入抽屉"），逆向策略恢复环境（如"从抽屉取出"）
- 两者交替执行形成自重置循环 τ_k = (τ→_k, τ←_k)，无需人工重置
- 逆向策略故意设计得比前向简单（恢复比操作容易），成功率达72%-86%
- Agent监控执行状态，判断成功/失败，决定是否继续循环或请求人工介入

#### 3. 部署时的过程监督与技能调度
- 部署时使用同一个VLM决策循环来编排技能
- Agent周期性查询环境摘要和机器人状态（通过MCP工具）
- 成功 → 更新任务记忆，进入下一子任务
- 失败 → 重试同一策略 / 切换到其他策略 / 重新规划
- 重复失败 → 升级至人工介入（Call Human工具）
- **关键优势**：收集和部署共享同一决策逻辑，减少语义和分布不一致

#### 4. 生命周期闭环学习
- 部署产生的轨迹可以回流到训练数据集D中
- 失败经验逐步转化为恢复策略，扩展策略库

### 底层VLA
- 使用 π₀.5 作为低层策略
- LoRA 微调（rank=16, alpha=16），bfloat16，batch=16，10k steps
- Flow matching objective + action chunking
- 推理使用 3 步（极少步数）

## 实验
- **平台**: AgiBot G01，双臂移动操作机器人，20 DoF + OmniPicker 夹爪
- **场景**: 4个真实环境（卧室梳妆台、厨房货架、书桌、便利店货架）
- **核心评测任务**: 梳妆台整理（长时序，4个子任务串联）

### 主要结果:

#### 数据收集效率
- 人工数据收集需要 2.16× 更多人力时间（相同数据量下）
- 人工在rollout中的介入需要 8.04× 更多（相比RoboClaw的自主收集）

#### 子任务策略迭代提升（EAP + 迭代rollout）
| 任务 | Iter 1 | Iter 5 | 提升 |
|------|--------|--------|------|
| Body Lotion | 42% | 86% | +44% |
| Primer | 46% | 80% | +34% |
| Lipstick | 4% | 46% | +42% |
| Tissue Wipe | 22% | 52% | +30% |
- 每次迭代增加50条数据，5次迭代后所有子任务大幅提升

#### 逆向重置策略成功率
- 4个任务的逆向策略：72%-86%成功率，足以维持自重置循环

#### 长时序任务
- RoboClaw 显著优于 Baseline 1（无框架的π₀.5）和 Baseline 2（4个子任务成功率的乘积）
- 改善来源：运行时监控 + 自动恢复，避免错误传播
- 相比baseline方法在长时序任务上提升25%成功率
- 减少53.7%人力投入

### 对比基线:
- Baseline 1: 直接使用π₀.5（同数据，无agentic框架）
- Baseline 2: 4个独立子任务成功率的乘积（理论期望值）
- 人工数据收集 baseline

### 失败分析
- **非退化失败**（Non-degrading）：环境状态未改变，可直接重试（如抓空）
- **退化失败**（Degrading）：环境状态被改变，需恢复策略（如物体倒了/滑走）
- 随着经验积累，恢复策略逐步加入策略库，减少人工介入需求

## 评价

### 优势
1. **全生命周期统一**：首次系统性地将数据收集、策略学习、长时序执行置于同一VLM决策循环中，解决了传统流水线的语义断裂问题
2. **EAP机制实用性强**：正-逆动作配对的思路简洁优雅，显著减少人工环境重置；逆向任务设计得更简单使得自重置循环可靠
3. **运行时监督有价值**：过程监督 + 动态恢复对长时序任务的稳定性提升显著（25%），验证了"VLM-as-supervisor"思路在真实部署中的有效性
4. **工程系统完整**：MCP工具接口、结构化记忆、失败分类与升级机制，形成了可落地的完整系统
5. **人力节省显著**：数据收集效率提升2.16×，rollout介入减少8.04×，总体减少53.7%

### 局限
1. **VLM推理延迟**：依赖云端大模型进行高层决策，延迟可能影响实时性
2. **逆向策略假设受限**：EAP要求每个操作都存在可行的逆操作，但很多真实任务不可逆（如切割、倒液体、粘合）
3. **评测规模有限**：仅4个子任务的梳妆台整理任务，长时序复杂度和多样性不够（vs π₀.5的10-15分钟家庭清洁）
4. **底层VLA依赖π₀.5**：框架本身不提升VLA能力，效果上界由底层VLA决定
5. **未与其他层次化方案严格对比**：如SayCan、HAMSTER、Hi Robot、π₀.5的内建层次化推理
6. **迭代数据增量较小**：每次迭代仅50条，缺少对更大规模数据积累效果的研究
7. **逆向策略成功率不完美**（72-86%），限制了长时间无人值守运行的可靠性

### 与已有工作的定位
- **vs π₀.5 的层次化推理**：π₀.5 是**模型内**层次化（统一模型同时做高层推理+低层控制），RoboClaw是**系统级**层次化（外部VLM做高层，独立VLA做低层）。前者更优雅，后者更灵活/模块化
- **vs SayCan**：都是"VLM/LLM规划 + 低层策略执行"，但RoboClaw增加了(1)运行时监控+恢复，(2)数据收集与执行的统一，(3)MCP工具接口
- **vs Genie Centurion**：Centurion的"rewind-and-refine"与EAP有相似精神（自动检测失败+重试），但RoboClaw的正-逆配对更系统化，且统一了收集和部署
- **vs Inner Monologue**：都强调运行时的反馈-重规划循环，RoboClaw进一步扩展到数据收集阶段

### 对VLA领域的贡献
1. **提出了"agentic robotics lifecycle"的系统范式**：强调数据收集、训练、部署不应割裂，应在统一语义下循环迭代
2. **EAP为自主数据收集提供了实用方案**：补充了VLA领域在"如何高效获取真实世界数据"方面的方法
3. **验证了VLM运行时监督对长时序任务的重要性**：quantitative evidence显示过程监督带来25%的成功率提升
4. **属于"VLM-as-Controller + VLA-as-Executor"系统设计的代表性工作**，为不具备训练端到端层次化VLA能力的团队提供了实用的替代方案
