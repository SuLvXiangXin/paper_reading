# RDT2: Exploring the Scaling Limit of UMI Data Towards Zero-Shot Cross-Embodiment Generalization

## 基本信息
- **作者**: Songming Liu*, Bangguo Li*, Kai Ma*, Lingxuan Wu*, Hengkai Tan, Xiao Ouyang, Hang Su, Jun Zhu
- **机构**: 清华大学
- **arXiv**: 2602.03310
- **年份**: 2025
- **项目主页**: https://rdt-robotics.github.io/rdt2/
- **方法线归属**: **VLM + Flow Head（跨具身泛化方向）**
- **前作**: RDT (Robotics Diffusion Transformer, 无 VLM 的 Transformer + Diffusion Head)

## 一句话总结
基于 Qwen2.5-VL-7B 的机器人基础模型，通过增强版 UMI 硬件采集 10,000+ 小时具身无关数据，三阶段训练（RVQ 离散化→Flow Matching→单步蒸馏）首次实现对未见具身/场景/物体/指令的"4U"零样本泛化，并在乒乓球、射箭等动态任务上展示实时反应能力。

## 核心贡献
1. **增强版 UMI 硬件**: 用尼龙66/玻璃纤维复合材料重新设计 UMI 夹爪，红外追踪替代 SLAM，提升鲁棒性和可扩展性
2. **10,000+ 小时开源数据集**: 覆盖 100+ 室内场景、3,000+ 物体、40+ 任务类型（家务/实验室），双阶段标注流水线 + Gemini 2.5 Pro 语言增强
3. **三阶段训练方案**:
   - **Stage 1 (RDT2-VQ)**: Qwen2.5-VL-7B + Residual Vector Quantization (RVQ) 将 0.8 秒 action chunk 压缩为 27 个离散 token，cross-entropy 训练；**同时通过 wds.RandomMix 混合 12M+ VQA 数据**（Ego4D, RoboVQA, RoboBrain 等），对齐语言-动作空间的同时防止 VLM 知识遗忘
   - **Stage 2 (RDT2-FM)**: 冻结 VLM 骨干，训练 400M 参数 RDT Action Expert（14层, 1024 hidden），Conditional Flow Matching 连续动作生成
   - **Stage 3 (RDT2-UltraFast)**: 单步一致性蒸馏，Stage 2 多步输出→单步推理，实现高频部署
4. **"4U" 零样本泛化**: 首次同时泛化到 Unseen embodiment + Unseen scene + Unseen object + Unseen instruction
5. **VQA 数据在 Stage 1 中混合训练（非独立阶段）**: 12M+ 视觉问答对（Ego4D, RoboVQA, RoboBrain 等）通过 `wds.RandomMix` 与 UMI 动作数据动态混采，共享 cross-entropy 目标，增强语义接地同时防止 VLM 知识遗忘

## 架构细节

### 模型配置
| 组件 | 层数 | Hidden Size | Heads | KV Heads | 参数量 |
|------|------|------------|-------|----------|--------|
| Qwen2.5-VL Backbone | 28 | 3584 | 28 | 4 | ~7B |
| RDT2 Action Expert | 14 | 1024 | 8 | 4 | ~400M |

### RVQ 动作 Token 化
- 将连续动作 chunk（0.8s, Ta=32 步）压缩为 27 个离散 token
- 多层残差量化：每层捕获上一层的残差，逐步逼近原始信号
- 优势：比 naive 均匀量化（如 OpenVLA 的 256 bins）保留更多精度，同时兼容 VLM 的 next-token prediction

### 三阶段渐进训练
```
Stage 1: VLM + RVQ tokens → 离散 next-token prediction (cross-entropy)
    ↓ 对齐语言-动作空间
Stage 2: 冻结 VLM → 训练 Action Expert (Flow Matching, 连续动作)
    ↓ 保留 VLM 语义 + 获得连续控制精度
Stage 3: 多步 FM → 单步蒸馏 (MSE regression)
    ↓ 实时推理
```

### VQA 预训练数据
- **Ego4D + QaEgo4D**: 大规模第一人称视频 VQA（情景记忆、时序定位）
- **HD-EPIC**: 厨房精细动作识别 + 3D 空间推理
- **RoboVQA**: 机器人长时序视频推理 + affordance reasoning
- **RoboBrain (ShareRobot)**: 100 万+ 任务规划/affordance/轨迹 QA
- **PixMo-Cap-QA, Cambrian-10M**: 互联网规模视觉语言数据

## UMI 数据集
- **规模**: 10,000+ 小时操作示教
- **场景**: 100+ 室内场景（家庭 + 实验室）
- **物体**: 3,000+ 不同物体
- **任务类型**: 40+ 类型
  - 家务类：擦拭(A038)、拾取(A039)、整理(A040) 等，每类 20-30 分钟连续操作
  - 实验室类：抓取原语、放置、堆叠等标准化操作
- **标注**: 双阶段流水线（任务级分割→精细动作分割）+ 结构化 schema（手+物体+动作原语）
- **语言增强**: Gemini 2.5 Pro 生成等价改写 + 简化版本，提升语言多样性
- **视觉**: 双目立体相机

## 实验结果

### 零样本评测（"4U" 协议）
- **具身**: Franka Research 3 + UR5e（训练从未见过的平台）
- **场景**: 3 个全新环境
- **物体**: 100+ 未见物体
- **指令**: 语言增强后的未见指令
- **任务**: Pick / Pick & Place / 排列 / 动态抓取 / 多步操作
- 每任务 256 次试验

### 微调评测
- **乒乓球**: 预测性规划补偿 1m/s 机器人速度限制，~100ms 反应时间
- **射箭**: 约 100ms 反应时间
- **织物折叠**: 泛化到新衣物
- **收桌子 (Table Bussing)**: 多步长时序任务，误差累积最小

### Baseline 对比
- **π0.5**: 标准 flow-based，Gemma-2B backbone + 300M action expert
- **π0-FAST**: FAST tokenizer + Gemma backbone
- RDT2 在 UMI 数据上预训练后，零样本和微调均优于两个 baseline

## 与现有方法的关系

### 与 RDT（前作）的关系
- RDT 是 "Transformer + Diffusion Head"（无 VLM），RDT2 升级为 "VLM + Flow Head"
- 继承了 RDT 的 Action Expert 设计思想，加入 Qwen2.5-VL-7B 作为骨干

### 与 π₀/π₀.5 的关系
- **相似**: 都是 VLM + Flow Matching 架构
- **区别**:
  - π₀ 用 PaliGemma/Gemma-2B 骨干，RDT2 用 Qwen2.5-VL-7B（更大、更强的视觉语言理解）
  - π₀ 数据来自多种机器人遥操作，RDT2 数据全部来自增强版 UMI（具身无关）
  - RDT2 增加了 Stage 1 RVQ 离散化预训练（语言-动作对齐），π₀ 直接 flow matching
  - RDT2 有 Stage 3 单步蒸馏实现实时推理

### 与 Being-H0.5 的关系
- **相似**: 都追求跨具身泛化
- **区别**: Being-H0.5 通过人手运动统一动作空间，RDT2 通过 UMI 具身无关硬件统一数据收集接口

### 与 DreamZero 的关系
- DreamZero 用视频世界模型（Wan2.1-14B）实现零样本泛化，RDT2 用 VLM + 大规模 UMI 数据实现
- 两者都强调"多样性数据"的重要性，但数据来源不同（DreamZero 多样非重复示教 vs RDT2 UMI 统一采集）

## 关键洞察
1. **UMI 作为跨具身数据统一接口**: 具身无关的硬件设计使得同一套数据可零样本部署到不同机器人，绕过了传统 VLA 需要每种机器人单独收集数据的瓶颈
2. **三阶段渐进训练的设计哲学**: 先离散化对齐语言-动作空间（借助 VLM 的 token prediction 能力），再用连续 flow matching 获得精度，最后蒸馏获得速度——每阶段解决一个问题
3. **RVQ 比 naive 量化更适合动作 token 化**: 残差量化逐层逼近，27 token 保留 0.8 秒高精度动作信息
4. **VQA 预训练增强语义接地**: 12M+ 视觉问答（含 Ego4D、RoboVQA 等具身相关 QA）使 VLM 更好地理解机器人操作场景
5. **数据规模与多样性是零样本泛化的关键**: 10,000+ 小时 + 100+ 场景 + 3000+ 物体的覆盖度是实现"4U"泛化的基础

## 局限性
- UMI 硬件仍限于夹爪操作，不支持灵巧手
- 零样本泛化仅在操作原语（pick, place 等）上验证，复杂长时序任务仍需微调
- 数据收集仍需大量人力（10,000 小时），虽然 UMI 降低了具身依赖但人力成本仍高
- RDT2-UltraFast 和完整数据集尚未开源（截至论文发布时）
