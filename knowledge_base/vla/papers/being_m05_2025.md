# Being-M0.5: A Real-Time Controllable Vision-Language-Motion Model (2025)

## 一句话总结
Being-M0.5 将 M0 的 motion foundation model 推进为实时可控 VL-motion model：基于 HuMo100M（5M motion sequences、100M instruction instances、part-level annotations）训练 7B VLMM，用 PRQ 部位级 motion tokenizer 和 HuMo263 表示实现随机指令、初始姿态、长时序、未见动作和部位控制，4-bit 推理可达实时，但仍需要 retargeting/低层控制器才能服务机器人。

## 问题
现有 VLMM / motion LLM 虽能做 text-to-motion，但不够实用：
- 对任意自然语言指令的响应弱，容易停留在固定 caption 模板。
- 多数模型默认标准初始姿态，难从随机 pose 继续生成合理 motion。
- 生成短片段为主，长时序动作缺乏一致性。
- 对训练集外动作泛化不足。
- 缺少对左/右臂、腿、躯干等身体部位的细粒度控制。
- 推理和后处理延迟高，难用于实时动画或 embodied interaction。

对 VLA 来说，这些问题对应"高层身体运动意图"到"可执行连续运动"之间的缺口；Being-M0.5 试图做一个可被交互式系统调用的 human motion layer。

## 方法
- 方法线归属: Vision-Language-Motion Model / Motion Generation Foundation Model（VLA-adjacent high-level motion prior）
- 核心 idea: 用 HuMo100M 提供大规模视觉-文本-运动三模态监督，并把人体运动拆成部位级 token，使 7B VLMM 可以实时生成、补全和控制长时序人体 motion。
- 关键技术点:
  - **HuMo100M**: 从 20M+ 视频中过滤 human-centric clips，经 YOLO tracking、WHAM SMPL 参数回归和 PHC 质量增强，形成 5M+ motion sequences、100M 多任务 instruction instances、paired visual clips、长时序拼接样本和 part-level 标签。
  - **层次化标注**: Gemini-1.5 Pro、PoseScript 和 rule-based posecodes 生成 whole-body、upper/lower body、limb-specific 描述，支持 "raise left arm" 这类部位级指令。
  - **VLMM 架构**: 基于 LLaVA-video-7B，包含 SigLIP 400M visual encoder、2-layer MLP projection、LLaMA-2-chat 7B backbone；slow-fast strategy 处理最多 64 帧视觉输入。
  - **训练课程**: motion-text alignment、vision-text-motion alignment、motion instruction tuning 三阶段训练；把 motion code 扩展到 LLM vocabulary，用 next-token prediction 同时处理文本和 motion token。
  - **多任务 controllability**: T2M、I2M、I2U、I2PM、I2LM、MPI、action-to-motion、motion reconstruction、motion-to-text，分别对应随机指令、未见动作、部位控制、长时序和随机初始姿态。
  - **PRQ（Part-aware Residual Quantization）**: 将身体分为 torso、left/right arm、left/right leg 五个 part，用共享 codebook 和 4 层 residual quantization 编码；相比整体 RQ，可独立控制身体部位，同时保留跨部位共享关节以减少不协调。
  - **HuMo263 + 实时解码**: 直接基于 SMPL 参数构造 HuMo263，避免 HM3D263 的 IK 后处理；frame-by-frame decoding 让 motion token 可以流式输出。4-bit llama.cpp 推理在测试 GPU 上保持 20+ FPS，H100 峰值 28.9 FPS。

## 实验
- Benchmark: HumanML3D、KIT-ML、HuMo-T2M、HuMo-I2M、HuMo-Unseen、HuMo-I2PM、HuMo-I2LM、HuMo-MPI、AMASS/3DPW prediction、UESTC action-to-motion、HumanML3D motion-to-text。
- 主要结果:
  - HumanML3D T2M: Being-M0.5-PRQ4 达到 FID 0.056、R@1 0.535、R@3 0.821、MMDist 2.865，优于 MotionGPT-v2、LMM、MoMask-RQ6 等。
  - HuMo-I2M: PRQ4 FID 0.148、R@1 0.428、R@3 0.625，相比 MotionGPT / MoMask 更适合真实自然语言 instruction。
  - HuMo-Unseen: 使用 HuMo100M 训练后 FID 8.65，显著优于 HumanML3D 训练的 65.04 和 Motion-X 训练的 43.28，说明大规模 web motion 对未见动作泛化关键。
  - Motion tokenizer: PRQ4 在 HuMo100M reconstruction 上 FID 2.317、MPJPE 38.06，优于 RQ-VAE8 的 3.526 / 64.56；共享关节和 part label 对 I2PM 消融显著。
  - Part control: 在 left-right exchange test 中成功率 76%，高于 ParCo 64% 和 T2M 46%；HuMo-I2PM 上 PRQ4 FID 1.831。
- 对比基线: MotionGPT、MotionGPT-v2、MotionLLM、LMM、MoMask、T2M-GPT、MLD、MotionDiffuse、ParCo、Fg-T2M++。

## 评价
- 优势: 从 M0 的"能生成大规模 motion"进一步走向"可实时、可控、可视觉条件化"；HuMo100M 把 part-level、long-form、visual-text alignment 放在同一个训练语料中；PRQ 是对 robot/VLA 也有启发的动作 token 设计，因为它把动作空间拆成有物理语义的可组合子空间；HuMo263 + frame-by-frame decoding 明确面向实时部署。
- 局限: 输出仍是人类 SMPL/HuMo263 motion，不是机器人关节命令或末端执行器 action；没有闭环 perception-action feedback、接触力、动力学稳定性、任务成功率或真实机器人 manipulation benchmark；要用于 humanoid/VLA 仍需 retargeting、whole-body controller、接触规划和安全约束；web video motion 与伪标签存在噪声；7B VLMM 虽可 4-bit 实时，但边缘部署和长上下文交互仍有成本。
- 对 VLA 领域的贡献: Being-M0.5 更接近一个可插入 embodied system 的高层 motion generator：VLA/robot policy 可以把它用作语言到全身动作草图、humanoid imitation prior、action chunk proposal、数据生成器或 teacher model，再由低层 policy 负责机器人动力学和环境交互。它补的是 motion prior 和 controllability，不替代 OpenVLA/π₀ 这类从视觉和本体感觉直接输出 robot actions 的策略模型。
- 与 Being-M0 的关系: M0 侧重 MotionLib/MotionBook 和 scaling law；M0.5 侧重 HuMo100M、多任务 instruction tuning、视觉输入、PRQ 部位控制和实时性。二者共同定义了 Being-M 系列在 embodied/VLA 知识库中的位置：不是 robot policy 主线，而是 humanoid motion foundation model 主线，可为未来 VLA 的高层动作表示和全身控制提供先验。
