# EMMA: Scaling Mobile Manipulation via Egocentric Human Data (2025)

## 基本信息
- 作者: Lawrence Y. Zhu, Pranav Kuppili*, Ryan Punamiya*, Patcharapong Aphiwetsa, Dhruv Patel, Simar Kareer, Sehoon Ha, Danfei Xu
- 机构: Georgia Institute of Technology
- arXiv: 2509.04443
- 链接: https://arxiv.org/abs/2509.04443, https://ego-moma.github.io
- 来源/验证状态: 已核对 arXiv v3 PDF 和项目页；project page 仍写 three real-world tasks，arXiv v3/RA-L 预印本写 four tasks 并加入 Push Chair，本文以 arXiv v3 为准

## 一句话总结
EMMA 将 egocentric human data 从桌面操作扩展到 mobile manipulation：用 Project Aria 采集人类移动操作，经过 differential-drive navigation retargeting、manipulation coordinate alignment 和 phase-aware control，与静态机器人操作数据 co-train，从而在无需 mobile teleoperation 数据的情况下接近或超过 Mobile ALOHA 风格遥操作基线。

## 问题
移动操作比固定桌面操作更难 scale：既要导航又要操作，mobile robot teleoperation 昂贵且场景受限。已有 human video transfer 多集中在 tabletop manipulation，无法直接提供移动底盘动作、导航上下文和导航/操作阶段切换。

## 方法
- 方法线归属: Human Data Pretraining for VLA / Egocentric human data co-training for mobile manipulation；不是通用大 VLA，而是端到端 imitation policy
- 核心 idea: 用人类 mobile manipulation 数据替代昂贵 mobile teleoperation 数据，让模型从 human demos 学导航和阶段结构，从 static robot data 学机器人手臂操作，再通过共享 ego vision trunk 把两者对齐
- 关键技术点:
  - 硬件: 人类和机器人都用 Project Aria ego view；机器人为 AgileX Tracer differential-drive base + 双 ViperX arms + wrist cameras
  - Navigation retargeting: 将人类 head trajectory 投影为 ground-plane waypoints，再优化成 differential-drive 可执行速度命令 `(v, omega)`，加入平滑和运动学约束
  - Manipulation alignment: 人类 hand motion 和机器人 end-effector action 都转到当前 ego camera frame，并按来源做 Z-score normalization
  - Co-training 架构: decoder-only Transformer，shared ego vision stem + shared trunk，human/robot proprio stems 和多 action heads；human batch 更新 navigation/head/manipulation human heads，robot batch 更新 robot manipulation head
  - Phase identification: 用 hand/head velocity ratio + GMM 无监督分段导航/操作阶段，推理时调制 action chunk，避免操作时底盘漂移和导航时手臂乱动

## 实验
- Benchmark: 真实移动双臂平台，Table Service、Handover Wine、Grocery Shopping、Push Chair；每个设置 50 trials
- 主要结果:
  - Handover Wine: 在相同 50 demos 静态 robot data 下，用 1 小时 human mobile data 达到 82% full success，比 1 小时 mobile robot teleop 的 Mobile ALOHA 52% 高 30pp
  - Scene generalization: 在仅 human demos 覆盖的新场景中，EMMA 达 54% success，Mobile ALOHA 几乎无法完成初始 grasp
  - Scaling: Handover Wine 中 human data 从 15min 到 60min，EMMA success 0.36→0.82；等时长 mobile teleop 数据的 Mobile ALOHA 0.26→0.52
  - Push Chair: 10min mobile robot data + 20min human data 可匹配 30min pure robot teleop，并优于分离 navigation/manipulation models
  - Ablation: 去掉 action retargeting success 下降约 30pp；去掉 phase identification 导致完整任务失败
- 对比基线: Mobile ALOHA with same HPT backbone, EMMA w/o action retargeting, EMMA w/o phase identification, separated navigation/manipulation models

## 评价
- 优势: 把 human egocentric 数据的价值从 manipulation 推到 navigation+manipulation 联合任务；明确区分 navigation retargeting、manipulation alignment 和 phase control 三个关键 gap；等采集时间下 human data 的边际收益高于 mobile robot teleop；真实移动双臂平台验证，不停留在仿真
- 局限: 仍需要静态 robot manipulation data，Push Chair 还需要少量 mobile robot data；假设部署视觉/空间分布接近 human demos；human navigation 对紧耦合 arm-base coordination 不够，需要机器人数据补齐；phase detection 是任务结构相关的工程模块，不是端到端涌现
- 对 VLA 领域的贡献: EMMA 是 Human Data Pretraining 路线向移动操作扩展的关键节点，说明人类 ego video 不只提供手部操作先验，也能提供导航路线、空间布局和任务阶段结构；它与 EgoVLA/EgoZero 的桌面路线互补，并为 EgoHumanoid 的 humanoid loco-manipulation co-training 提供直接前身
