# MEM: Multi-Scale Embodied Memory for Vision Language Action Models (2025)

## 基本信息
- 作者: Marcel Torne*, Karl Pertsch*, Homer Walke, Kyle Vedder, Suraj Nair, Brian Ichter, Allen Z. Ren, Haohuan Wang, Jiaming Tang, Kyle Stachowicz, Karan Dhabalia, Michael Equi, Quan Vuong, Jost Tobias Springenberg, Sergey Levine, Chelsea Finn, Danny Driess
- 机构: Physical Intelligence, Stanford University, UC Berkeley, MIT
- 链接: https://pi.website/research/memory

## 一句话总结
提出多尺度记忆系统 MEM，通过高效视频编码器实现短时视觉记忆（秒级，处理遮挡和上下文适应）+ 语言摘要机制实现长时语义记忆（分钟级，追踪任务进度），使 VLA 能在保持实时推理的前提下完成长达 15 分钟的复杂操作任务。

## 问题
现有 SOTA VLA（如 π₀.6）大多仅基于当前单帧观测做决策，缺乏记忆能力。但真实世界的复杂多阶段任务要求机器人同时具备：
1. **短时记忆**：解决自遮挡、理解运动动力学、从失败中快速调整抓取策略
2. **长时记忆**：记住已完成的子任务步骤、已处理的物体、门/柜是否需要关闭

**核心矛盾**：将完整观测历史输入策略在长时序任务中不可行（推理延迟随帧数爆炸增长），但简单压缩又会丢失关键信息。**没有单一模态能同时满足两种记忆需求**——图像适合精细短时记忆但长序列开销大，语言适合语义压缩但丢失空间细节。

## 方法
- **方法线归属**: VLM + Diffusion/Flow Head（基于 π₀.6），新增**多尺度记忆**维度，同时涉及**层次化 VLA**（高层/低层分解）
- **核心 idea**: 将记忆问题按时间尺度分解为两种模态——用视频编码器高效压缩短时视觉记忆，用自然语言摘要追踪长时语义记忆，两者互补覆盖秒级到分钟级的记忆需求。

### 关键技术点:

#### 1. 多尺度记忆架构（高层/低层分解）
- **分解公式**: π(a_{t:t+H}, l_{t+1}, m_{t+1} | o_{t-T:t}, m_t, g) ≈ π_LL(a | o_{t-K:t}, l_{t+1}, g) · π_HL(l_{t+1}, m_{t+1} | o_t, m_t, g)
- **低层策略 π_LL**：基于短时观测序列 (K ≪ T) + 子任务指令 l + 全局目标 g，输出连续动作
- **高层策略 π_HL**：基于当前观测 + 上一步语言记忆 m_t + 全局目标 g，输出子任务指令 l_{t+1} 和更新后的语言记忆 m_{t+1}
- **关键创新**：相比 π₀.5 的层次化分解，MEM 的高层策略**额外预测和更新语言记忆 m_{t+1}**，实现记忆的自主管理

#### 2. 语言记忆机制（长时记忆）
- **设计**：m_t 是一段自然语言摘要，描述已完成的语义事件
- **训练数据生成**：给定带子任务标注的轨迹 + 每步成功/失败标记，用预训练 LLM 生成记忆转换 m_t → m_{t+1}
- **压缩策略**（关键）：指示 LLM 丢弃/压缩不再需要的信息（如将"放了浅绿碗、深蓝碗和亮黄碗"压缩为"放了三个碗"），保持记忆简洁
- **优势**：
  - 减少训练-推理分布偏移：当策略重复失败时，记忆不会累积重复的子任务指令（vs "naive" 拼接方案）
  - 保持推理速度：压缩后 token 数量可控
- **对比 naive 拼接**：直接拼接所有历史子任务指令作为记忆，在推理时因策略重复失败导致分布偏移，性能显著更差

#### 3. 视频编码器（短时记忆）
- **问题**：逐帧编码再输入 VLA backbone 的方式，推理延迟随帧数线性增长，4帧以上就超过 300ms 实时阈值
- **设计**：扩展标准 ViT 为视频编码器
  - 每 4 层交替应用空间注意力（帧内，双向）和时间注意力（跨帧，因果掩码）
  - 时间注意力：同一 patch 位置跨时间步做注意力，因果掩码确保未来帧不影响过去
  - 复杂度：O(Kn² + nK²)（vs naive joint attention O(n²K²)），其中 n=patch数, K=帧数
- **Token 压缩**：仅传递当前时间步的 patch 表示给 VLA backbone，丢弃过去帧的 token → 与单帧 VLA 的 token 数量一致
- **初始化**：**不引入新可学习参数**，仅修改注意力模式 + 添加固定正弦时间位置编码（t=0 时编码为 0，保证 K=1 时与原始 VLM 完全一致）
- **效率**：16帧输入仅需约 300ms（vs naive 方案约 4s），满足实时约束

#### 4. 预训练+后训练的记忆扩展
- **预训练**：6帧（5帧历史 + 当前），1秒步幅 → 约 5 秒记忆
- **后训练**：可灵活扩展到 18帧、54 秒视觉记忆（类似 LLM context window 扩展）
- **本体感知记忆**：将历史关节角度通过线性投影编码为连续 token（而非文本形式），K帧仅产生 K个 token

#### 5. 集成到 π₀.6
- **VLM backbone**: Gemma3-4B（SigLIP 400M + Gemma 4B）
- **动作生成**: FAST action token + flow matching action expert (860M)，梯度不回传到 VLM backbone
- **输入**: 448×448 分辨率，最多 4 路相机
- **推理**: 使用 RTC / Training-time RTC 实现异步实时推理

## 实验

### A. 长时记忆任务（核心实验）
- **Recipe Setup**：机器人从厨房各处取出菜谱所需食材/器具，放到指定位置，关闭柜门。42个菜谱训练，5个未见菜谱+未见厨房评测。需追踪已取物品、记住关柜门。
- **Clean Kitchen**：擦拭台面、放好食物、洗碗、挂碗架。需记忆已清洁区域、是否已加肥皂、碗的正反面是否都洗了。
- **结果**：π₀.6 无记忆几乎无法完成；MEM 显著提升（任务进度从约 30% 提升到约 70%+）

### B. 上下文适应（In-Context Adaptation）
- **Chopstick Pick Up**：桌面高度 OOD 导致频繁抓空，记忆使策略调整抓取高度 → +62% 成功率
- **Open Fridge**：开门方向不明确，记忆使策略在失败后切换方向 → +11% 成功率
- **训练方式**：收集人类干预修正数据，将失败尝试保留在短时记忆中训练

### C. 记忆能力基准测试
- **任务**: Swap 3 Mugs（部分可观测）、Find Object（记忆物体位置）、Unpack Groceries（记忆剩余物体）、Scoop Coffee（计数）、Grilled Cheese（计时）、Window Cleaning（空间记忆）
- **对比基线**:
  - **Pool Memory** (类似 ContextVLA [Jang et al.]): 逐帧编码→平均池化为单个 memory token
  - **Proprio Memory** (类似 TA-VLA [Zhang et al.]): 仅历史本体感知状态
  - **No Memory**: π₀.6 baseline
  - **MEM-Posttrain-Only**: 仅后训练引入视频编码器（不预训练记忆能力）
- **结果**:
  - Pool Memory: 简单记忆任务有效，长时序/多物体追踪退化严重
  - Proprio Memory: 仅在需要记住自身状态的任务有效，无法记忆环境状态
  - MEM 是唯一在**所有**记忆能力上都表现优秀的方法
  - 预训练记忆能力至关重要：MEM-Posttrain-Only 性能显著下降（即使初始化自同一 π₀.6 检查点）

### D. 灵巧操作基准
- **任务**: Table Bussing, Shirt Folding, Clean Up Counter, Make Bed, Dishes to Sink, Batch Folding, Box Building
- **结果**: MEM 匹配无记忆 π₀.6 的 SOTA 性能，**未出现因加入记忆导致的性能退化**（prior work 常报告的 causal confusion 问题）
- **原因**: 多样化预训练数据（不同最优性、速度、控制频率 + 互联网视频）防止了视频编码器的虚假相关

## 评价

### 优势
1. **准确定位核心问题**：不是简单地"给 VLA 加更多帧"，而是深入分析了长/短时记忆的本质需求差异，用不同模态分别解决
2. **视频编码器设计优雅**：不引入新参数、保持单帧兼容性、从预训练 VLM 权重初始化——工程上极其实用
3. **语言记忆的压缩机制**：解决了 naive 历史拼接的分布偏移问题，这是一个实际部署中常被忽略的重要问题
4. **预训练记忆能力**：证明在多样数据上预训练记忆比仅在目标任务后训练更有效，类似 VLM 预训练的意义
5. **无 causal confusion**：大规模多样化预训练数据自然缓解了记忆策略常见的因果混淆问题，避免了需要额外辅助损失
6. **上下文适应是附赠能力**：短时视觉记忆自然带来了从失败中调整策略的能力

### 局限
1. **语言记忆依赖子任务标注**：训练数据需要子任务级语言标注 + LLM 生成摘要，标注成本较高
2. **高层策略的准确性**：长时序任务中，高层策略的子任务预测和记忆更新一旦出错可能导致级联失败
3. **记忆上限**：视觉记忆最长 54 秒，语言记忆虽无硬上限但信息会累积丢失；论文自身提出未来需扩展到跨 episode 的持续记忆（周/月/年）
4. **评测主要在 Physical Intelligence 自有平台**：未在标准仿真 benchmark 上验证，可复现性受限
5. **未与 latent memory 方法对比**：如 Sam2Act、MemoryVLA 等使用潜在记忆状态的方法未被纳入对比

### 对 VLA 领域的贡献
1. **首次系统性解决 VLA 的多尺度记忆问题**：明确提出并验证了"短时视觉+长时语言"的混合模态记忆架构
2. **将 π₀.5 的层次化推理扩展为层次化记忆**：从"高层预测子任务"升级为"高层同时管理语义记忆"，是层次化 VLA 的重要演进
3. **解锁 VLA 的新能力类别**：上下文适应（in-context adaptation of manipulation strategies）是此前无记忆 VLA 不具备的
4. **证明记忆不会损害性能**：在灵巧操作任务上匹配无记忆 SOTA，打破了"加记忆必降性能"的领域偏见
5. **15 分钟连续操作**：将 VLA 端到端执行任务的时间跨度推向新高

### 与 π₀/π₀.5 的关系
MEM 是 π₀.5 的直接演进（基于 π₀.6 实现）。π₀.5 引入了层次化推理（高层预测子任务+低层生成动作），MEM 在此基础上增加了两个关键维度：
- **短时维度**：高效视频编码器替代单帧输入，使低层策略具备视觉记忆
- **长时维度**：高层策略从"仅预测子任务"升级为"预测子任务+更新/管理语言记忆"
- π₀.5 的上下文窗口受限 → MEM 通过模态分离将有效记忆扩展到 15 分钟

### 与其他记忆方法的关系
| 方法 | 记忆类型 | 时间跨度 | 限制 |
|------|---------|---------|------|
| 密集帧历史（Octo, BET） | 图像序列 | ~数秒 | 推理延迟爆炸 |
| Pool Memory (ContextVLA) | 池化压缩图像 | ~十几秒 | 丢失精细细节 |
| Proprio Memory (TA-VLA) | 本体感知状态 | ~分钟 | 无环境记忆 |
| Keyframe (BPP, MeMeR) | 关键帧选择 | ~分钟 | 丢失动力学信息 |
| Point Tracks (TraceVLA) | 2D点轨迹 | ~十几秒 | 丢失精细空间信息 |
| Latent Memory (Sam2Act) | 隐状态 | ~十几秒 | 仅短时验证 |
| Language Memory (OneTwoVLA) | 纯语言 | ~分钟 | 丢失空间/抓取细节 |
| **MEM (本文)** | **视频+语言混合** | **~15分钟** | **需子任务标注，语言记忆有损** |
