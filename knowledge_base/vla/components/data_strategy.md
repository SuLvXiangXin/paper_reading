# 数据策略

## Cross-embodiment 跨具身训练
混合不同机器人形态的数据联合训练，使策略学到通用的操作知识。
- 关键挑战：不同机器人的动作空间维度不同（零填充对齐）、运动学差异
- 代表工作：OXE dataset, Octo, RT-X
- **Octo的方案**：手动筛选 OXE 25 个数据集（800k 轨迹），按数据集大小加权 + 多样性翻倍 + 重复数据降权；统一 gripper 动作表示（+1=开, 0=合）；缺失相机零填充
  - 关键发现：更多数据集显著提升性能（25 > 11 > 1），大 shuffle buffer (500k) 对多数据集训练至关重要
- **π₀的方案**：7种机器人构型统一到18维动作空间（最大维度），低维机器人零填充；按n^0.43加权不同task-robot组合
- **π₀.5的发现**：跨具身数据（ME + CE）对泛化贡献巨大，即使来自完全不同形态的机器人

## Co-Fine-Tuning（RT-2 首创）
**核心思想**：在 VLM 微调为 VLA 时，同时保留原始互联网视觉-语言数据，而非仅用机器人数据微调。

### RT-2 的 Co-Fine-Tuning（2023，首创）
- **动机**：防止灾难性遗忘，保留 VLM 预训练的抽象视觉概念
- **机制**：每个训练 batch 混合机器人数据和互联网 VL 数据，通过采样权重控制比例
- **数据配比**：RT-2-PaLI-X 机器人数据约 50%；RT-2-PaLM-E 约 66%
- **互联网数据来源**：WebLI（10B 图文对过滤到 1B）+ 多种 VQA/captioning 数据集
- **消融验证（55B 模型，未见任务泛化）**：
  - co-fine-tuning: **63%**
  - fine-tuning only: 52%（-11%）
  - from scratch: 9%（-54%）
- **关键洞察**：co-fine-tuning 使策略同时暴露于抽象视觉概念（互联网）和低层动作（机器人），两者互相增强

### OpenVLA 的缺失（2024）
- OpenVLA 仅在机器人数据上微调，**未采用 co-fine-tuning**
- 导致语义泛化弱于 RT-2-X（在 BridgeV2 语义泛化评测中 RT-2-X 更强）
- 一个重要教训：纯机器人数据微调会部分遗忘 VLM 预训练知识

### 后续发展
- **RT-2-X**: 继承 co-fine-tuning，扩展到跨具身 OXE 数据
- **π₀.5**: 将 co-training 思想大幅扩展为 5 类异构数据源（机器人 + 高层语义 + 网络数据 + 口头指令 + 跨具身），是 RT-2 co-fine-tuning 的全面升级

## Pre-training / Post-training 配方
**由 π₀ 首次系统引入机器人领域，类比 LLM 的预训练→对齐范式。**
**Octo 是先驱**：首次在 GRP 中验证大规模预训练 + 少量目标域微调（~100 demos）的有效性，为后续工作奠基。

### Octo 的配方（2024，先驱）
- **预训练**：800k OXE 轨迹，25 数据集，多种机器人/传感器/任务
- **微调**：~100 条目标域示范，50k 步，统一超参，单 GPU <5h
- **发现**：微调全部参数优于冻结子集；统一配方跨场景通用

### π₀ 的配方（2024）
- **预训练阶段**：
  - 数据：10,000+ 小时，自有 903M 步 + OXE/Bridge/DROID（占9.1%）
  - 目标：广泛能力、从错误恢复的能力
  - 语言标签：任务名称 + 细粒度段标注（~2秒子轨迹）
- **后训练阶段**：
  - 数据：高质量任务特定数据（5-100+ 小时不等）
  - 目标：流畅高效的任务执行
- **互补逻辑**：预训练教恢复（包含各种非最优行为），后训练教策略（高质量示范）
- **实验验证**：仅预训练 → 能做但不够好；仅后训练 → 脆弱不能恢复；两者结合最佳

### DreamZero 的"多样性优先"配方（2026，WAM 独特范式）
DreamZero 提出了与传统 VLA 截然不同的数据收集哲学——**多样性优先于重复性**：
- **预训练数据**: ~500 小时 AgiBot G1 遥操作，22 个真实环境
- **与传统 VLA 数据的关键区别**:
  - 每集 ~4.4 分钟、~42 个子任务（远长于典型 VLA 数据集的单任务短集）
  - **任务淘汰机制**：每任务收集 50 集后 deprecated，迫使操作者持续提出新任务→长尾分布
  - 粗粒度任务标注（如"整理物品"）而非精细任务定义
  - **3-task episode 结构**：每集连续执行 3 个任务，学习任务过渡
- **消融验证**: 500h 多样数据 >> 500h 重复数据（50% vs 33% task progress）
- **为什么 WAM 能利用多样数据而 VLA 不能**:
  - VLA 学"观测→动作"直接映射，需要密集重复数据覆盖状态-动作空间
  - WAM 的视频预测目标继承自视频预训练，只需额外学习 inverse dynamics
  - Inverse dynamics 需要多样的状态-动作对应关系，重复数据反而不利
- **VLA 在多样数据上完全失败**: 14B VLA 在相同多样数据上 0% task progress（包括放大到 14B 的 VLA），说明这是方法范式差异而非规模问题
- **意义**: 如果机器人数据不需要重复示教，数据收集成本可大幅降低——操作者只需在真实环境中自由执行有用行为

### EgoScale 的三阶段配方（2026，Human Data 扩展）
在 π₀ 的 pre-train/post-train 基础上，EgoScale 引入了一个关键的中间阶段——**aligned mid-training**——形成三阶段管线：
- **Stage I (Human Pretraining)**: 20,854 小时 egocentric 人类视频，全参数解冻，256 GPU, batch=8192
  - 数据：in-the-wild ego 视频 + EgoDex（高精度），覆盖 9,869 场景、6,015 任务、43,237 物体
  - 动作监督：相对 SE(3) wrist motion + 22-DoF retargeted hand joints
  - 无 proprioception → learnable placeholder token
- **Stage II (Aligned Mid-training)**: 50 小时人类 + 4 小时机器人对齐数据，冻结 VLM backbone，仅更新 vision encoder + DiT
  - **核心创新**：人类演示与机器人使用匹配的相机配置和 motion-capture stack，实现视觉和动作层面的精确对齐
  - **作用**：将 Stage I 学到的抽象操作结构"落地"到机器人的感知和控制空间
- **Stage III (Post-training)**: 任务特定的 20-100 条机器人 demo，10K steps
- **关键洞察**：Scale 和 Alignment 可以且应该解耦——Stage I 提供多样性和语义基础，Stage II 提供具身精确对齐
- **与 π₀ 的区别**：π₀ 的预训练数据是机器人数据（同域），EgoScale 的预训练数据是人类数据（跨域），因此需要额外的 mid-training 阶段来弥合具身差距

### π₀.5 的配方（2025，进一步发展）
将co-training扩展到5类异构数据源，远超传统VLM数据保留：
1. **MM（移动操作数据）**：目标平台的直接经验
2. **ME（多环境非移动数据）**：不同具身形态但场景多样
3. **CE（跨具身实验室数据）**：任务多样但环境简单
4. **HL（高层子任务预测）**：教模型任务分解
5. **WD（网络数据）**：图像描述、VQA、物体定位，提供广泛语义知识

**关键实验发现**：
- 预训练阶段97.6%数据非目标平台，但对最终泛化至关重要
- ME和CE去除任一都显著降低性能
- WD主要影响OOD物体识别和高层推理
- 口头指令（VI，仅占11%高层数据）对高层策略极为关键
- 仅用目标平台数据直接训练（不做异构co-training）即使包含测试环境数据也表现不佳

## 交互式数据收集（Interactive / Human-in-the-loop Data Collection）🆕
> 代表作：RoboCopilot (2024, CoRL)、GCENT/Genie Centurion (2025, AgiBot)

### 核心思想
**被动示范 vs 交互式示范的本质区别**：
- **被动示范（Offline BC）**：人类预先演示完整轨迹，策略从静态数据集学习。协变量偏移（covariate shift）导致策略在测试时的状态分布不同于训练时，误差持续累积。
- **交互式示范（DAgger 系列）**：策略在线执行，人类仅在策略失败时介入并提供修正动作。介入数据天然覆盖策略的失败模式，从根本上缓解协变量偏移。

### "collect-then-deploy" vs "deploy-then-collect" 范式对比
| 范式 | 数据收集时机 | 效率 | 数据质量 | 代表工作 |
|------|------------|------|---------|---------|
| **Collect-then-deploy** | 策略部署前 | 人机比≤1:1 | 均匀覆盖，大量冗余"简单段" | ALOHA、传统遥操作 |
| **Deploy-then-collect** | 策略部署后 | 可做到 1:N | 聚焦真实失效点，数据效率高 | RoboCopilot、**GCENT** |

**关键洞察**（GCENT 的观察）：策略失败往往集中于特定"高冲击决策点"——抓取最后动作块、精密对准等，而非均匀分布在整个轨迹上。全轨迹遥操作收集大量冗余的"接近阶段"数据，但这些数据对策略改进贡献极小。

### HG-DAgger 框架（Human-Gated DAgger）
- **基本流程**：热启动（offline warmup）→ 策略部署 → 人类按需介入 → 更新数据集 → 微调策略 → 循环
- **关键设计**：由人类（而非自动判断）决定何时介入，更灵活可靠
- **与原始 DAgger 的区别**：原始 DAgger 需要专家为策略轨迹的每一步重新标注，在高维操作任务中不可行；HG-DAgger 只要求人类实时接管，更实用

### RoboCopilot 的工程贡献（2024，CoRL）
RoboCopilot 解决了 HG-DAgger 在双臂操作落地的关键工程障碍——**遥操作系统不支持安全的在线接管**：
- **外骨骼/GELLO 方案的问题**：策略执行期间领导臂静止，人类接管时领导-跟随臂存在位姿偏差，产生跳变和安全隐患
- **RoboCopilot 的解决方案**：双边控制使领导臂在策略执行时实时跟踪机器人状态；人类接管仅需握住并移动装置，无任何同步延迟或跳变
- **双边力反馈**：人类感受到机器人的接触力，更直觉地理解机器人状态，提升介入数据质量
- **量化结果**：移动搬运任务 91.7% vs 58.3%（+57.1%），长时序任务首次达到完整流程通过

### GCENT 的扩展贡献（2025，AgiBot）🆕
GCENT 在 RoboCopilot 的基础上进一步解决了**可扩展性（1:N 多机器人）**问题：

#### 1. 倒带（Rewind）机制
- 系统维护 3 秒关节状态缓冲；失败时反向回放恢复历史状态
- **价值**：避免从零重置，直接在失效点附近扩充关键状态空间覆盖
- **阶段性策略**：
  - 早期（低成功率）：直接干预优于倒带——模型需学习从失败中恢复的能力（0.69 vs 0.50）
  - 后期（高成功率）：倒带优于直接干预——消除错误序列，提升首次成功轨迹质量（0.91 vs 0.84）

#### 2. Task Sentinel（任务哨兵）
- 基于 InternVL 2.5-2B + MLP 二值分类头：输入图像+指令 → 输出当前步成功/失败
- **检测成功而非失败**：成功定义清晰稳定，失败类型随训练阶段动态变化难以穷举
- **核心作用**：解放人类从全程监控转为按需介入，实现 1:N 多机器人监督

#### 3. 可扩展性数据
| 策略成功率 | 单人双机效率 |
|---------|------------|
| 40% | 1.86 （理论最大 2.0）|
| 60% | 1.90 |
| 80% | 1.92 |

#### 4. 数据效率
- 相同帧数下：GCENT 平均提升 **40%** 成功率
- 相同性能水平下：GCENT 仅需对比方法 **44.5%** 的帧数

### 两种训练模式对比
| 模式 | Continual DAgger | Batched DAgger |
|------|-----------------|----------------|
| 训练时机 | 实时在线微调 | 收集完数据后从零重训 |
| 优势 | 快速反映当前失败模式，人类能及时看到改进 | 避免灾难性遗忘，训练分布稳定，性能最优 |
| 劣势 | 灾难性遗忘，非平稳训练分布 | 不能实时反馈，需等待重训完成 |
| 推荐实践 | 用 Continual DAgger 收集高质量交互数据，再用 Batched DAgger 从零重训 |

### 与其他数据质量提升方案的综合对比
| 方案 | 核心机制 | 人力需求 | 适用场景 | 扩展性 |
|------|---------|---------|---------|--------|
| **RoboCopilot HG-DAgger** (2024) | 双边遥操作无缝接管，覆盖失败分布 | 需实时专家监督 | 需覆盖分布外失败模式 | 单机器人 |
| **GCENT** (2025) | 倒带机制+Task Sentinel，仅关键点干预 | 按需介入（Task Sentinel 辅助） | 大规模实采，长时序/精密任务 | 1:N 验证 |
| **RoboClaw EAP** (2026) | 正-逆动作对自重置，减少人力 | 较低（可批量自动重置） | 大规模自动化数据循环 | 系统级 |
| **ADC 对抗数据收集** | 人为注入扰动，增加分布多样性 | 需操作者在场制造扰动 | 快速提升初始数据多样性 | 单机器人 |
| **MEM 上下文适应** | 策略从短时记忆的失败修正中在线学习 | 训练时需收集失败→修正数据对 | 执行期在线适应 | — |
| **DreamZero 多样性优先** | 多样非重复数据，WAM 自然从长尾分布泛化 | 低（无需覆盖所有状态） | WAM 方法线 | — |

### 交互式数据收集的演进脉络
```
DAgger (2011, Ross et al.): 理论奠基，专家重标注所有状态
  → SafeDAgger / LazyDAgger: 降低人类负担（安全预测/切换成本）
  → HG-DAgger (2018): 人类作为门控函数，按需接管
  → EnsembleDAgger / ThriftyDAgger: 不确定性/预算感知介入
  → Fleet-DAgger / SIRIUS-FLEET (2022/2024): 多机器人调度
  → RoboCopilot (2024): 双边遥操作硬件, HG-DAgger 落地双臂操作
  → GCENT / Genie Centurion (2025): 倒带机制 + Task Sentinel, 1:N 可扩展, 真机 VLA 精调
  → RoboClaw (2026): VLM 元控制器全生命周期统一
```

### 对 VLA 数据收集的启示
- **相同轨迹数下，交互数据 >> 被动数据**：这对昂贵的 VLA 数据收集具有直接指导意义
- **数据质量集中度胜过覆盖广度**：失效点的数据比均匀全轨迹数据价值更高
- **Task Sentinel 提供了运行时策略评估范式**：区别于 VQA 式成功检测，逐帧二值分类（成功检测）更适合在线监控
- **倒带机制有助于状态空间集中覆盖**：可在不增加总数据量的情况下显著提升关键状态区域的数据密度

## Cross-Embodiment Transfer via Video-Only Data（视频跨具身迁移）
> DreamZero (2026) 提出的新范式

### 核心思路
WAM 的跨具身迁移不需要动作标注——仅用其他具身/人类的视频数据做视频预测目标训练，即可增强 target 机器人的任务泛化。

### DreamZero 的两种设置
1. **Robot-to-Robot Transfer（YAM → AgiBot G1）**: 20 min YAM 视频 → 未见任务 +45% relative improvement
2. **Human-to-Robot Transfer**: 12 min 人类 egocentric 视频 → 未见任务 +42% relative improvement
3. **Few-shot Embodiment Adaptation**: AgiBot 预训练 → 30 min YAM play data → 保持零样本泛化

### 与传统跨具身方法的区别
- **π₀/π₀.5**: 需要跨具身动作数据（带动作标注）
- **Being-H0.5**: 统一动作空间映射所有具身的动作表示
- **DreamZero**: 仅需 video-only 数据→视频预测目标→增强 world model → implicit IDM 自动受益
- **关键优势**: 无需动作标注→可利用大规模互联网人类视频→数据获取成本极低

### 理论基础
WAM 将策略分解为 video prediction（已由视频预训练提供）+ inverse dynamics（从视频到动作的映射）：
- Video-only 跨具身数据增强了视频预测模型对任务动态的理解
- Implicit IDM 利用增强的视频预测做更准确的动作推理
- 本质上是"视觉经验"的迁移，而非"运动经验"的迁移

## Human Data as Pretraining Source（人类数据作为预训练源）
> 2024-2026 年兴起的重要数据策略，利用大规模人类视频为机器人策略提供可扩展的监督信号

### EgoScale 的关键发现（2026）
- **Scaling Law**: 人类数据的 action prediction validation loss 与数据规模呈 log-linear 关系：L = 0.024 - 0.003·ln(D)，R² = 0.9983
- **离线 loss 预测真机性能**: validation loss 与 downstream task completion 强相关，可作为训练质量的可靠代理指标
- **无饱和迹象**: 在 1K-20K 小时范围内，性能随数据规模单调增长
- **意义**: 首次为"需要多少人类数据"提供了定量预测工具

### 动作表示选择（EgoScale 消融）
- **Retargeted joint-space（最优）**: 22-DoF 人手 keypoints → 机器人手关节空间，保留精细手指信息
- **Fingertip-based（次优）**: SE(3) fingertip trajectories + MLP mapping，小误差易导致不合理关节配置
- **Wrist-only（最差）**: 仅 6-DoF wrist motion，缺失手指级别的监督，在精细操作任务上严重失败
- **结论**: 灵巧操作必须保留手指级别的动作监督，wrist-only 不够

### 与机器人数据预训练的互补关系
- **π₀/π₀.5**: 在大规模机器人数据上预训练 → 直接 post-training
- **EgoScale**: 在大规模人类数据上预训练 → aligned mid-training → post-training
- **DreamZero**: 视频模型预训练 → 机器人视频+动作联合训练 → 跨具身 video-only 数据增强
- **互补而非替代**: 人类数据提供操作行为的多样性和规模，机器人数据提供具身精确性；两者结合最优

## Co-Fine-Tuning → 异构 Co-Training 演进线
```
RT-2 co-fine-tuning (2023): 机器人数据 + 互联网VL数据（2类）
  → RT-2-X (2023): + 跨具身OXE数据（3类）
    → π₀.5 (2025): 5类异构数据全面co-training
    → EgoScale (2026): 人类数据预训练 + 对齐mid-training + 任务post-training（解耦范式）
    → DreamZero (2026): 视频预训练 + 多样非重复机器人数据 + video-only跨具身增强（WAM范式）
```

## 口头指令 (Verbal Instruction)
- **新型数据收集范式**：人类专家实时用语言指导机器人完成任务
- 本质是对"好的高层决策"的示范，收集成本远低于遥操作
- 首次由π₀.5引入并验证有效性
- 可视为连接人类常识知识与机器人策略的桥梁
