# NavGPT: Explicit Reasoning for VLN with LLMs (2023)

## 基本信息
- **标题**: NavGPT: Explicit Reasoning in Vision-and-Language Navigation with Large Language Models
- **作者**: Gengze Zhou, Yicong Hong, Qi Wu
- **机构**: 阿德莱德大学
- **arXiv**: 2305.16986
- **年份**: 2023 (AAAI 2024)
- **引用数**: 328
- **方法线归属**: **LLM 推理驱动**

## 核心贡献
1. **首个将 LLM 用于 VLN 显式推理的工作**（零样本，无需 VLN 训练数据）
2. **ReAct 范式应用于导航**：Thought → Action → Observation 循环
3. **揭示 LLM 的导航高层规划能力**：指令分解、子目标规划、常识推理、异常处理
4. **揭示核心瓶颈**：视觉→文本的信息损失，而非推理能力

## 方法架构

### 视觉感知文本化
- 每个视点获取 3×8=24 张自我中心视图（FoV 45°, 8 方向×3 俯仰）
- **BLIP-2** (ViT-G FlanT5XL) 对每张图生成 caption
- 同方向 3 张 caption 由 **GPT-3.5** 合并为一句
- **Faster-RCNN** 检测物体 + 深度信息 → 3 米内物体的方位角和距离
- 以 agent 朝向为"前方"，8 方向描述拼接

### GPT 推理链（ReAct 范式）
- 每步输出：Thought（推理）→ Action（选择 viewpoint ID）→ Observation（环境反馈）
- Thought 不触发环境交互，作为 bootstrap 推理
- 推理功能：指令分解、进度跟踪、子目标规划、异常处理

### Prompt 设计
- **System Principle**：任务定义 + 方位约定 + 行为约束 + 推理格式
- **历史管理**：用 GPT-3.5 将历史观测压缩为一句话（控制 prompt 长度）
- **当前观测**：完整描述（8 方向的场景+物体+可导航点）

## 实验结果

### R2R Val-Unseen（GPT-4 零样本）
| 方法 | SR↑ | SPL↑ | NE↓ |
|------|------|------|------|
| **NavGPT (零样本)** | **34** | **29** | **6.46** |
| DuET (未训练) | 1 | 0 | 9.74 |
| HAMT (有监督 SOTA) | 66 | 61 | 2.29 |
| DuET (有监督 SOTA) | 72 | 60 | 3.31 |

- 零样本 SR 34%，远超未训练 DuET（1%）
- 但与有监督 SOTA（72%）差距 38 个百分点

### 消融实验（GPT-3.5，216 样本）

| 消融 | SR | 结论 |
|------|-----|------|
| FoV@60 12 views | 10.19 | 太粗 |
| FoV@30 36 views | 13.89 | 太细 |
| **FoV@45 24 views** | **16.67** | **最优** |
| 仅 BLIP-2 caption | 11.11 | 信息不足 |
| + 物体检测 | 15.97 | +4.86 |
| + 物体 + 深度距离 | 16.67 | +5.56 |

## 关键洞察

1. **感知是瓶颈，推理不是**：零样本失败主要因视觉→文本信息丢失，不是 LLM 推理能力不足
2. **LLM 具备空间意识**：GPT-4 能从文本历史绘制俯视轨迹图
3. **历史管理是通用难题**：压缩丢失 vs prompt 长度限制
4. **深度/距离信息的边际贡献明确**：防止过早停止
5. **定位价值**：不以 SOTA 为目标，而是揭示 LLM 在导航中的能力和瓶颈

## 局限性
1. **视觉→文本信息瓶颈**：BLIP-2 caption 的粒度直接决定上限
2. **历史摘要信息丢失**：进度跟踪出错
3. **高成本高延迟**：每步需调用 GPT-4 + GPT-3.5 + BLIP-2 + RCNN
4. **仅离散环境**
5. **零样本天花板**：SR 34% 实际应用不够

## 与其他工作的关系
- **后续**: NavGPT-2（升级为 VLM 直接视觉输入，SR 大幅提升）
- **影响**: 开创 LLM-based VLN，328 引用
- **被继承**: NaVid 论文中验证了 text-based 历史完全失败（0% SR），印证 NavGPT 的瓶颈分析
