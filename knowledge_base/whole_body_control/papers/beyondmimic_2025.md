# BeyondMimic: From Motion Tracking to Versatile Humanoid Control via Guided Diffusion

- **arXiv**: 2508.08241
- **年份**: 2025（arXiv v4: 2025-11-13）
- **机构**: UC Berkeley + Stanford University
- **作者**: Qiayuan Liao†*, Takara E. Truong†, Xiaoyu Huang†, Yuman Gao, Guy Tevet, Koushil Sreenath‡, C. Karen Liu‡（†共同一作，‡共同通讯）
- **引用数**: 119
- **方法线归属**: **Diffusion-based WBC / Motion Tracking + Guided Diffusion**

## 一句话总结

紧凑RL运动跟踪 + 潜在扩散模型 + 分类器引导，零样本泛化到任意下游任务。

## 解决的问题

现有人形机器人控制方法面临两大核心矛盾：**自然性 vs. 通用性**的鱼与熊掌问题。DeepMimic 系列虽能实现高质量运动跟踪（跳跃、武术），但每个运动需要单独调参（motion-specific tuning），无法扩展到数百个技能。AMP（Adversarial Motion Priors）学习了运动"风格"，但策略不可跨任务复用，每个新任务必须从头重训。层次控制（PHC/HOVER 路线）通过将高层规划器与底层跟踪器分离来实现复用，但规划器与控制器的训练解耦导致 mismatch，同时牺牲了运动的灵活性和自然感。VAE 类生成模型（如 ASE）虽然支持技能组合，但依赖训练时的显式目标条件，对隐式目标（障碍物规避、长时导航）泛化能力差，产生分布外的抖动运动。BeyondMimic 旨在同时解决：以统一配方高质量学习数百个技能，并在测试时零样本泛化到训练中从未见过的任务。

## 核心方法

### 整体架构（两阶段流水线）

```
阶段一: 多样化人类动作 → 运动跟踪RL策略（每个动作独立策略，共享超参数）
                                      ↓
阶段二: VAE 蒸馏（DAgger） → 潜在状态-动作扩散模型
                                      ↓
测试时: 分类器引导（task-specific cost）→ 零样本下游任务泛化
```

训练完全在仿真中进行（MuJoCo/Isaac Gym），使用厂商标准参数，无额外系统辨识（system identification）。部署框架用 C++ 实现，优化实时执行，解决 sim-to-real 中的时延问题。两个阶段均零样本迁移到真实硬件，无需针对运动或机器人的单独调整。

### 运动跟踪框架（阶段一：Scalable Motion Tracking）

**核心设计原则**：一套配方（one recipe fits all）——每个运动用独立策略，但共用同一套 MDP 公式和超参数，不针对任何动作单独调整。

**锚体重锚（Anchor-Centered Tracking）**：将目标体的姿态表示从全局坐标系转为相对锚体（通常为 root/torso）的 yaw 对齐、高度保持（yaw-aligned, height-preserving）坐标系。锚体本身跟随参考运动的全局位置，非锚体的期望姿态通过变换 A(T_b_ref, T_anchor) 计算。这一设计允许扰动和 sim-to-real 误差引起的"良性全局漂移"，同时保留运动风格（style），显著提升鲁棒性和迁移性。

**简洁奖励设计**：任务奖励 r_task 对所有目标体进行位置误差、姿态误差、线速度误差、角速度误差的均匀加权跟踪，每项用高斯指数映射（Gaussian-shaped exponential）。仅使用 3 个正则化惩罚：关节限位惩罚（r_limit）、动作平滑惩罚（r_smooth）、自碰撞惩罚（r_contact）。无手工设计的接触力惩罚、滑动惩罚、跌倒惩罚等大量启发式项。

**观测空间**：无时序堆叠（no temporal stacking），包含运动相位 ψ（仅作进度提示）、锚体姿态误差 e_anchor（R9，含位置+Rot6D方向）、IMU twist Vimu、关节状态（θ-θ₀, θ̇）、上一时刻动作 a_last。使用 Rot6D 旋转表示而非四元数或轴角，消融实验证明这对 sim-to-real 至关重要（四元数版本跟踪误差更高甚至失败一次）。

**自适应采样（Adaptive Sampling）**：长动作序列中不同片段难度差异极大，均匀采样会过采样容易片段、欠采样困难片段。采用基于经验失败率（empirical failure rate）的自适应采样，优先训练高失败率片段，随着策略收敛逐渐恢复均匀采样。消融显示：没有自适应采样时，4个运动中有3个在困难片段（如翻筋斗）失败，自适应采样对简单运动也将训练迭代减半（2k vs. 4k）。

**关节阻抗选择**：避免高阻抗（高PD增益）——高阻抗虽然简化控制（接近运动学回放），但在真实硬件上会放大传感器噪声、降低冲击吸收的被动顺应性，并掩盖关节状态和指令中隐含的扭矩信息。论文提供了在高自由度人形系统上选择合理关节阻抗和动作比例的启发式流程（见 Supp. S1）。

**域随机化**：保持最小域随机化集合：地面摩擦系数和恢复系数、默认关节位置 θ₀（模拟标定误差）、躯干质心位置，以及随机速度扰动。刻意避免随机化扭矩扰动、时延等，保持控制目标的纯粹性。

### 统一潜在扩散模型（阶段二：Latent Diffusion Model）

**两阶段 LDM 训练**：

*第一步 VAE（DAgger 蒸馏）*：训练条件变分自编码器，编码器接收参考运动的相位 ψ 和锚体误差 e_anchor，输出潜在表示 z = E(ψ, e_anchor)；解码器结合 z 和当前本体感知（重力向量 g, Vimu, θ, θ̇, a_last）重建动作 â = D(z, [...])。用 DAgger（数据集聚合）训练，损失为 ELBO：重建动作误差 + β·KL 散度。这样的编码方案比直接编码原始 PD 动作更稳定，因为运动状态比原始动作更结构化。

*第二步 State-Latent 扩散模型*：滚动已训练 VAE 采集状态-潜在轨迹 τ = [s_{t-N}, z_{t-N}, ..., s_t, z_t, ..., s_{t+H}, z_{t+H}]，用 DDPM 框架训练去噪网络 z_φ(τ_k, k)。轨迹包含 N 步历史 + 当前帧 + H 步未来（0.64 秒预测域），采用混合 character-yaw-centric 参数化确保空间一致性。每个时间步的状态和潜在分别使用独立的去噪步数 k_s, k_z，允许为观测和未来姿态设置不同噪声级别，这是支持运动填补的关键设计。

**选择 LDM 而非标准扩散模型的动机**：（1）动作空间（PD setpoints）高度不规则，含尖锐扭矩毛刺，直接在动作空间做扩散训练不稳定，违反扩散模型的平滑假设；（2）大型扩散网络推理延迟不可忽略，而 LDM 的轻量级解码器可利用最新观测产生精确、动力学一致的动作。消融结果：无潜在编码的基线在仿真中翻筋斗成功率仅 5%，LDM 达到 95%，且此高性能成功零样本迁移到真实机器人。

**状态参数化**：根部姿态 (p_imu, R_imu) 和速度 (v_imu, ω_imu) 相对于当前时刻 t 的根部坐标系表示；体位置 p_b 和速度 v_b 相对于各自时刻的局部根坐标系表示。

### 分类器引导（Classifier Guidance）——核心创新点

**原理**：扩散模型不像 GAN/VAE 那样学习显式的数据分布，而是学习数据分布的**梯度场（score function）**：∇_τ log p(τ)。利用贝叶斯规则可将无条件 score function 转为条件形式：

```
∇_τ log p(τ|τ*) = ∇_τ log p(τ) + ∇_τ log p(τ*|τ)
```

其中 τ* 为期望/最优轨迹。通过将条件似然近似为可微任务代价函数 G(τ)（p(τ*|τ) ∝ exp(-G(τ))），条件梯度简化为：

```
∇_τ log p(τ*|τ) = -∇_τ G(τ)
```

这使得扩散过程可在每次去噪迭代中，根据任意可微代价函数进行梯度下降，**无需重训**即可优化新目标。实际上是在每个去噪步骤中额外执行一步梯度更新，将采样轨迹向目标方向引导。

**为何 State-Action 联合建模是关键**：任务目标定义在状态空间（"到达 waypoint"、"避开障碍物"），而典型策略输出动作空间，两者之间需要前向动力学模型桥接（计算量大，实时性差）。BeyondMimic 通过在 LDM 中**联合建模状态和动作**，隐式捕获动作如何影响未来状态，在推理时直接在状态上定义代价并反向传播，避免了显式前向动力学模型。这是一个预测控制（predictive control / receding-horizon control）的框架：每个决策时刻预测未来轨迹，最小化代价后取当前时刻动作执行。

**具体任务代价函数**：
- *速度跟踪*：期望速度与预测根部速度之间的 L2 误差
- *Waypoint 导航*：期望位置与预测根部位置之间的距离
- *障碍物规避*：由 SDF（有符号距离场）定义的距离代价，在每个去噪步提供全预测域的距离梯度
- *运动填补（Motion Inpainting）*：对 0.2s 间隔的稀疏未来关键帧，设置关键帧位置对应时刻的预测状态为约束

**多目标组合**：不同任务代价可直接相加，无需重新调参大量的正则化项（这是与模型based和学习based方法相比的核心优势）。例如 Waypoint + SDF 代价的直接求和实现了场景感知导航，joystick + SDF 代价组合使普通遥操作也能自动避障。

## 实验结果

### 运动多样性与质量

训练了约 2.5 小时多样化人类运动数据，在高保真仿真中验证全部运动，将 30 个代表性片段（共 15 分钟）部署到真实机器人。覆盖从静态/平衡类（单腿站立、各姿势起立）到高度动态类（单腿跳跃、旋转跳、180°/360°翻转踢、空中翻筋斗）再到风格化/表达性类（老年人步态、舞蹈、体育动作），在单一配方和共享超参数下全部成功。

### 人类级敏捷性

在户外（软土、落叶、不平地面，训练中未见）部署复杂杂技序列，包括需要爆发性起跳、受控空中旋转和精确落地的空中翻筋斗。空中阶段：机器人峰值加速度达 **31 m/s²**，骨盆角速度最高 **20 rad/s**（均值 7.01 rad/s），与训练有素人类空中动作（平均 7.75 rad/s）相当。落地干净，几乎无需恢复，表明精确的空中姿态控制。

### 人类感知自然度

用力感应跑步机对比机器人和人类走路/跑步的地面反力（GRF）曲线（按体重归一化），机器人 GRF 形状与人类高度相似：步行呈双峰（明显的脚跟触地峰和蹬地推进峰），奔跑呈单峰，加载/推进时序一致。步行时机器人力峰更尖锐，归因于缺少趾关节（限制了过渡和蹬地阶段的顺应性），奔跑时该差异较小。

用户研究（N=77）：被试对比 BeyondMimic vs. Unitree 原生控制器（该机器人当前 SOTA）的 20 对 5 秒步行/奔跑视频片段，选择"更像人、更自然"的一方。BeyondMimic 整体被显著偏好（**70.8% vs 29.2%**，p<0.001，Cohen's h=0.859），步行 57.0% vs 43.0%（p<0.001，h=0.281），奔跑 **84.7% vs 15.3%**（p<0.001，h=1.532）。

### 零样本泛化任务表现

- **速度跟踪**：仿真中步行速度跟踪误差 12.14%，奔跑 13.65%
- **Waypoint 导航**：从多个起始点平滑稳定到达目标
- **遥操作**：全向步行，抗大扰动（踢腿），可持续奔跑 50m+
- **多模态步态**：低速命令自然产生步行或轻松慢跑两种模态，无需显式指定；速度命令增大时自发产生步行→奔跑平滑过渡
- **运动填补**：从操纵杆控制步行状态，注入 0.2s 间隔翻筋斗关键帧，扩散策略平滑过渡进入翻筋斗，完成后平滑返回步行；连续 3 个翻筋斗中间穿插步行/奔跑的长时域执行
- **避障**：将 waypoint 代价 + SDF 障碍物代价组合，机器人成功绕行障碍物到达目标

### 消融实验关键发现

| 设计选择 | 影响 |
|---------|------|
| 旋转表示：Rot6D vs 四元数/轴角 | 四元数误差更高，轴角偶发失败；Rot6D 显著更好 |
| 时序历史观测 | 加入历史反而降低性能（最小域随机化下，历史使策略过拟合仿真特定模式） |
| 仿真机械参数精度（以 armature 为例） | 忽略 armature 导致关节过度加速和自碰撞 |
| 部署时延 | 2ms 时延即增加速度误差，5ms 导致1次失败，10ms 2/3次失败 |
| 自适应采样 | 无自适应采样时 4/4 中 3 个运动在困难片段失败 |
| 潜在扩散 vs. 直接动作扩散 | 直接扩散翻筋斗成功率仅 5%，LDM 为 95% |

## 实现细节

**机器人平台**：Unitree 人形机器人（具体型号未在文中明确，根据 Unitree native controller 对比推测为 G1 或 H1）。

**RL 训练**：MDP 时间不变（time-invariant），策略为 MLP，无时序堆叠观测。奖励：r_task（4项均匀加权跟踪误差的高斯指数之和）- λ_l·r_limit - λ_s·r_smooth - λ_c·r_contact。动作：归一化关节位置 setpoint θ_sp = θ₀ + α⊙a，发送给底层电机 PD 控制器。物理引擎：同一套参数在 MuJoCo 和 IsaacGym 均可训练（通用性验证）。

**VAE 架构**：编码器输入 [ψ, e_anchor]（参考运动组件），输出潜在 z；解码器输入 [z, g, Vimu, θ, θ̇, a_last] 重建动作。DAgger 训练。

**LDM 架构**：Denoiser 为 Transformer 架构（见 Fig. 7B），处理 state-latent 轨迹 τ。预测域 H = 0.64 秒，历史长度 N 步。详细超参数见 Supp. S3。

**部署框架**：完全用 C++ 实现，优化实时执行，仅运行于 CPU + 低端移动 GPU。实现了策略推理与硬件执行的精确同步，解决 sim-to-real 中时延导致的 gap。

**数据**：约 2.5 小时多样化人类动作，全部未标注（task-agnostic），无任何人工标签。已开源，被 MJLab 和 Unitree RL Lab 等公共 RL 仓库作为默认方法采用。

## 零样本泛化能力

BeyondMimic 展示了 5 类训练中从未见过的下游任务的零样本泛化：

1. **速度命令跟随（Joystick Teleoperation）**：以期望线速度和偏航角速度为输入，通过速度代价引导扩散，产生全向步行。在强外部扰动（踢腿）下快速恢复并继续追踪。

2. **Waypoint 导航**：以期望位置为目标，通过位置代价引导扩散，从多个初始位置规划平滑轨迹到达目标。

3. **运动填补（Motion Inpainting）**：注入稀疏未来关键帧（0.2s 间隔），由扩散填充过渡和中间帧，实现翻筋斗等高动态技能的在线插入和技能切换。关键帧条件作为扩散步的 inpainting 约束（对应时刻的状态设为已知，对这些步不去噪或只部分去噪）。

4. **障碍物规避（Scene-aware Navigation）**：基于 SDF 的障碍物代价与 waypoint 代价相加，机器人自动绕障。即使在 joystick 遥操作模式下结合障碍物代价，也能在用户控制偏离目标时自动缓解碰撞。

5. **任务组合（Task Composition）**：多任务代价自由相加，支持组合测试中从未枚举过的目标组合，无需任何重训或微调。

**关键优势**：与基于 VAE 的方法（如 ASE）相比，分类器引导不依赖训练时的目标条件，无需预先定义所有可能任务，特别适合隐式或难以在训练时指定的目标（障碍物规避、长时导航）。与基于模型的方法相比，无需繁琐的正则化和行为塑形项，简单的任务代价即可触发适当行为。

## 与相关工作的关系

| 方法 | 类别 | 与 BeyondMimic 的区别 |
|------|------|----------------------|
| **DeepMimic** (2018) | 单运动 RL | 需要针对每个运动的 motion-specific 调参，不可泛化 |
| **AMP** (2021) | 对抗运动先验 | 学习运动风格但策略不可跨任务复用，每任务需重训 |
| **PHC/PHC+** (2023) | 全身运动控制 | 单技能专门策略，无任务泛化能力 |
| **MaskedMimic** (2024) | 掩码引导动作合成 | VAE/条件生成，训练时需显式目标条件，隐式目标泛化差 |
| **HOVER** (2024) | 多模策略蒸馏 | 层次控制导致规划器-控制器 mismatch，牺牲敏捷性 |
| **ASE** (2022) / **CALM** | VAE 技能组合 | 依赖显式 goal conditioning，OOD 时产生抖动运动 |
| **Diffusion Policy** (Robotics) | 扩散动作策略 | 仅建模动作空间，缺乏前向状态预测，无法做状态层面引导 |

BeyondMimic 的核心差异点：
- vs. PHC/HOVER：不是层次控制，而是**单一扩散模型**同时处理跟踪和泛化
- vs. VAE 类方法：不依赖训练时目标条件，分类器引导在测试时完成，天然 open-ended
- vs. 仅动作扩散策略：**联合建模状态和动作**，状态代价可直接反向传播无需前向动力学
- vs. 常规 MPC：扩散模型本身作为强力运动先验，简单任务代价即可触发复杂自然行为，无需大量约束

## 局限性与未来工作

1. **状态估计依赖**：扩散模型的生成质量继承自底层本体感知（proprioception）精度，传感器噪声直接传播到生成轨迹，LDM 的潜在空间提供一定鲁棒性但仍受限。未来方向：传感器融合或学习型估计器。

2. **预测域限制（0.64 秒）**：足以应对反应式控制和局部障碍物规避，但不足以支持需要提前预判的长时规划任务（远处目标、需早期规避的障碍物）。

3. **历史依赖 vs. 响应性**：历史观测对稳定未来预测至关重要，但会导致模型陷入重复运动模式，即使引导激活也如此。当前通过增大引导权重反制，但过大引导权重在模式切换或高方差状态时会不稳定扩散过程，导致运动开始/结束时踉跄。未来方向：探索减少历史依赖的方案。

4. **细粒度任务目标**：引导优化在粗粒度目标（速度、位置）表现好，细粒度目标效果下降，且需要轻量级手动调整引导权重。未来方向：监督微调（SFT）和 adapter 风格控制层（类似视觉域的 ControlNet 等方法），实现无需重训的细粒度可组合轨迹控制。

5. **探索性 RL 不足**：阶段一多技能联合策略（类似 HOVER）由于探索不充分会产生不自然行为，而单独策略虽质量高但规模成本高（每运动一个策略），这是当前设计的折中。
